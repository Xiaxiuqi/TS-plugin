# 项目文档

## 2026-01-24 更新日志 (v1.4.2)

### 优化：头像编辑弹窗交互与适配
1.  **点击关闭逻辑**：
    -   现在点击“头像选择弹窗”和“头像裁剪弹窗”的遮罩层（空白区域）可以直接关闭弹窗。
    -   在裁剪弹窗中，点击遮罩层关闭等同于取消操作。
2.  **窄屏适配修复**：
    -   优化了头像选择和裁剪弹窗的 CSS 定位逻辑。
    -   将 `position` 修正为 `fixed !important` 并配合 `flex` 布局，确保弹窗在任何分辨率下都能垂直居中显示。
    -   增加了 `z-index: 10020`，确保弹窗层级高于编辑面板。
    -   限制了弹窗最大高度并添加了滚动条，防止在高度受限的屏幕上内容被截断。
    -   [紧急修复] 修复了点击头像弹窗遮罩层会导致底层面板（如编辑面板）意外关闭的问题（添加了 `e.stopPropagation()`）。
    -   [紧急修复] 修正了弹窗居中逻辑，移除多余的 `margin: auto`，严格遵循 Flexbox 布局，确保在各种分辨率下正确居中。

### 修复：技能面板关闭状态同步问题
1.  **问题**：点击技能面板右上角的关闭按钮时，虽然面板视觉上关闭了，但内部状态 `state.isSkillsOpen` 未更新为 `false`，且未移除物品仓库按钮的激活状态。导致无法再次打开技能面板，且按钮状态显示错误。
2.  **原因**：通用的面板关闭事件处理器（绑定在 `body` 上）缺少对 `panelType === 'skills'` 的处理逻辑，且该处理器阻止了事件冒泡，导致绑定在 `document` 上的专用处理器未能执行。
3.  **修复**：在通用的关闭按钮事件处理器中补充了针对 `skills` 面板的逻辑：
    -   正确设置 `state.isSkillsOpen = false`。
    -   检查 `state.isInventoryOpen` 状态，仅当物品面板也关闭时才移除按钮激活状态。

## 2026-01-14 更新日志 (v1.4.0)

### 新增功能：技能面板编辑支持
1.  **技能面板编辑按钮**：在技能面板头部添加了编辑按钮，与其他面板风格统一。
2.  **编辑模式集成**：`toggleEditMode` 函数现在会同步切换技能面板的编辑状态。
3.  **技能卡片编辑交互**：
    -   在编辑模式下，技能卡片显示绿色编辑遮罩和图标。
    -   点击卡片不再显示详情，而是打开编辑弹窗。
4.  **技能编辑弹窗**：
    -   支持编辑技能名称、类型、归属、描述。
    -   支持编辑动态详情字段（排除标准字段）。
5.  **数据保存逻辑**：
    -   实现了 `saveSkillData` 函数，支持将修改后的技能数据回写到数据库表格。
    -   使用 `_src.rowIdx` 精确定位数据行。
    -   支持 Extension Settings 和 LocalStorage 双重保存机制（通过 `performDirectInjection`）。

### 修复与优化
-   **UI一致性**：技能面板的编辑体验现在与物品仓库完全一致。
-   **代码复用**：复用了物品编辑的逻辑结构，确保稳定性。

## 2026-01-14 更新日志 (v1.2.4)

### 修复：全技能视图分类标签切换问题
1.  **问题**：在“全技能视图”中，点击分类标签无法正确切换激活状态，导致无法筛选技能。
2.  **原因**：原有的 DOM 操作逻辑在处理类名移除时存在作用域问题。
3.  **修复**：重构了 `renderAllSkillsView` 函数。
    -   使用 jQuery 对象创建元素，替代原有的字符串拼接，提高代码安全性和可维护性。
    -   使用 `$this.siblings('.ci-inv-cat-tab').removeClass('active')` 确保互斥逻辑正确执行。
    -   使用 `.data('cat', cat)` 绑定数据，避免属性解析错误。

## 2026-01-14 更新日志 (v1.2.3)

### 新增功能：技能面板 (Skills Panel)
1.  **双面板切换**：
    -   在物品仓库按钮检测到包含“技能”关键词的表格时，点击按钮会弹出二级菜单。
    -   二级菜单包含“物品仓库”和“技能面板”两个选项。
    -   两个面板互斥显示，状态栏按钮高亮逻辑同步更新。
2.  **技能数据提取**：
    -   在 `processData` 中增加了对技能表的识别逻辑。
    -   支持提取：技能名、类型、拥有者、描述及其他动态列。
    -   数据存储在 `state.cachedData.allSkills` 中。
3.  **技能视图**：
    -   **角色视图**：按拥有者分组显示技能，支持分页。
    -   **技能视图**：按技能类型（如“攻击”、“被动”等）分组显示，支持点击标签筛选。
4.  **技能详情与使用**：
    -   点击技能图标显示详情弹窗。
    -   **使用功能**：点击“使用”按钮，自动生成叙事文本并填入输入框。
        -   主角使用：生成“主角使用了[技能名]”。
        -   NPC使用：生成“主角和[NPC]商量了一下，让[他/她]使用了[技能名]”。
        -   支持自动识别主角名称和NPC人称代词。

### 代码优化
-   **模块化渲染**：将技能面板的渲染逻辑拆分为 `renderSkillsPanel`, `renderCharacterSkillsView`, `renderAllSkillsView`。
-   **通用组件**：复用了物品卡片的样式和交互逻辑，保持 UI 风格一致。
-   **状态管理**：新增 `state.isSkillsOpen` 状态，用于控制面板显示和交互。
