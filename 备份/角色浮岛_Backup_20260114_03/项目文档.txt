更新日志 v1.36 (修复固定面板尺寸限制与最小宽度):
======================================

### 1. 修复固定/拖拽后面板无法拉大的问题 [已完成] ✓

**功能需求**
- **背景**：用户反馈当面板被“固定”或拖拽到远离浮岛的位置后，依然受到之前计算的 `max-width` 限制（基于浮岛位置计算的剩余空间），导致无法手动拉大面板宽度。
- **目标**：当面板处于“固定”状态或用户开始手动拖拽面板时，应清除之前设置的 `max-width` 限制，允许用户自由调整面板尺寸。

**代码实现**
- `src/角色浮岛/index.ts`:
  - **Pin 按钮逻辑**：在点击固定按钮（`.ci-pin-btn`）时，如果切换为固定状态，立即清除面板的 `max-width` 样式（`$panel.css('max-width', '')`）。
  - **Drag 逻辑**：在面板头部拖拽开始时（`handleDrag`），立即清除面板的 `max-width` 样式。这确保了用户在移动面板后，不再受到之前位置的挤压限制。

### 2. 设置面板最小宽度 [已完成] ✓

**功能需求**
- **背景**：用户反馈面板在被挤压时可能会被压缩成一条线，影响使用。

**代码实现**

  - 为所有面板类（`#ci-panel`, `#ci-map-panel`, `.ci-sub-panel`）添加 `min-width: 200px !important;`。
  - 使用 `!important` 确保该样式优先级高于 JS 动态设置的 `width` 或 `max-width`（在极窄情况下）。

**自检确认**
- [x] 面板在被固定后，可以自由拉大宽度，不受浮岛位置限制。

更新日志 v1.35 (修复面板宽度弹回问题):
======================================

### 1. 修复面板从边缘拉远时宽度自动弹回的问题 [已完成] ✓

**功能需求**
- **背景**：用户反馈虽然面板在屏幕边缘会被挤压缩窄，但一旦将面板拉离边缘，宽度会自动弹回原来的大尺寸。此外，在窄屏模式下，由于 `width` 属性仍然保持较大的数值（仅视觉上被 `max-width` 限制），导致拖拽调整宽度的功能失效。
- **目标**：确保当面板被挤压时，其实际 `width` 属性也会同步更新为较小的值。这样当面板离开边缘时，会保持被挤压后的宽度，而不会弹回。同时解决窄屏下拖拽失效的问题。

**代码实现**
- `src/角色浮岛/index.ts`:
  - 修改 `syncPanelPosition` 函数：
    - 在设置 `max-width` 之后，增加一步检查：`if ($pan.outerWidth() > availableW) { $pan.css('width', availableW + 'px'); }`。
    - 这确保了当可用空间小于当前宽度时，面板的 `width` 属性会被强制更新为可用空间的大小。

**自检确认**
- [x] 面板被边缘挤压时，`width` 属性会同步变小。
- [x] 将面板从边缘拉远后，宽度保持在被挤压后的大小，不会自动变宽。
- [x] 在窄屏挤压状态下，拖拽右下角调整宽度（向内缩小）功能正常。

======================================

### 1. 数据库模板存储位置适配 [已完成] ✓

**功能需求**
- **背景**：数据库脚本（数据库神V10.ts）更新了模板的存储位置，从浏览器的 `localStorage` 迁移到了 SillyTavern 的 `extensionSettings`（具体命名空间为 `shujuku_v100__userscript_settings_v1`）。
- **问题**：原有的地图模板注入/删除功能依赖于直接读写 `localStorage` 中的特定 key，导致在更新后的数据库脚本环境下功能失效。
- **目标**：修改 `src/角色浮岛/index.ts`，使其能够正确读取和写入新的存储位置，同时保留对旧版 `localStorage` 的兼容性（以便在旧版数据库脚本环境下仍能工作）。

**代码实现**
- `src/角色浮岛/index.ts`:
  - **新增辅助函数**:
    - `getDatabaseTemplate_ACU(key)`: 尝试从 SillyTavern 的扩展设置中读取模板。如果找不到，回退到 `localStorage` 读取。支持新版 Profile 格式的 key（如 `shujuku_v100_profile_v1__<SLOT>__template`）和旧版 key。
    - `saveDatabaseTemplate_ACU(key, value)`: 尝试将模板保存到 SillyTavern 的扩展设置。如果写入失败（例如 API 不可用），回退到 `localStorage`。同时会触发 `saveSettingsDebounced` 以持久化更改。
  - **逻辑替换**:
    - `forceDeleteMapTemplate`: 替换了原有的 `localStorage` 遍历和删除逻辑，改用辅助函数处理。
    - `injectMapTables`: 替换了原有的 `localStorage` 读取和写入逻辑，改用辅助函数处理。现在会优先读取新位置的模板，注入地图表后，保存回原位置。
    - `removeMapTables`: 替换了原有的 `localStorage` 读取和写入逻辑，改用辅助函数处理。

**自检确认**
- [x] `getDatabaseTemplate_ACU` 能正确读取 `extensionSettings` 中的新版模板。
- [x] `saveDatabaseTemplate_ACU` 能正确写入 `extensionSettings` 并触发保存。
- [x] 兼容性：如果 `extensionSettings` 不可用或 key 不存在，能回退到 `localStorage`。
- [x] 地图注入功能（`injectMapTables`）不再硬编码依赖 `localStorage`。

更新日志 v1.2.12 (物品再利用生命周期重构):
======================================

### 物品再利用生命周期重构 [已完成] ✓

**功能优化需求**
- **严谨的解锁逻辑**：用户保存设置后，不再立即解锁功能。必须等待 LLM 实际生成了对应的物品（在物品列表中检测到该物品名）后，功能才正式解锁。
- **自动重置逻辑**：如果功能已解锁，但检测到对应物品消失（例如被再次销毁或丢弃），功能应自动重置（清除名称、选项、激活状态），回归初始锁定状态。

**状态机设计**
1. **初始/重置态 (Locked)**: 无名称，无选项，未激活。显示锁图标。
2. **等待态 (Waiting)**: 用户已保存名称和选项，但物品尚未生成。显示锁图标（保持锁定直到物品出现）。
3. **激活态 (Active)**: 检测到物品存在。显示自定义名称，无图标，功能可用。

**代码实现**
- `src/角色浮岛/index.ts`:
  - `STORAGE_RECYCLE_ACTIVE_KEY`: 新增布尔值存储键，用于标记“是否已正式激活”。
  - `handleRecycleSave`: 仅保存名称和选项，**不再**直接修改 DOM 或播放解锁动画，也不设置 ACTIVE 状态。
  - `renderInventoryPanel`: 承担核心状态管理职责。
    - **检查**: 遍历物品列表查找自定义名称。
    - **状态流转**:
      - 找到物品 -> 设置 `ACTIVE = true` -> 渲染解锁状态。
      - 没找到物品 & `ACTIVE` 为真 -> 判定为物品丢失 -> 清除所有数据（重置） -> 渲染锁定状态。
      - 没找到物品 & `ACTIVE` 为假 -> 判定为等待生成 -> 保持数据但渲染锁定状态。

- [x] 物品出现后，按钮自动刷新为解锁状态。
- [x] 物品消失后，按钮自动重置为初始锁定状态，且清除已保存的设置。
