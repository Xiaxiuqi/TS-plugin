更新日志 v1.2.12 (物品再利用生命周期重构):
======================================

### 物品再利用生命周期重构 [已完成] ✓

**功能优化需求**
- **严谨的解锁逻辑**：用户保存设置后，不再立即解锁功能。必须等待 LLM 实际生成了对应的物品（在物品列表中检测到该物品名）后，功能才正式解锁。
- **自动重置逻辑**：如果功能已解锁，但检测到对应物品消失（例如被再次销毁或丢弃），功能应自动重置（清除名称、选项、激活状态），回归初始锁定状态。

**状态机设计**
1. **初始/重置态 (Locked)**: 无名称，无选项，未激活。显示锁图标。
2. **等待态 (Waiting)**: 用户已保存名称和选项，但物品尚未生成。显示锁图标（保持锁定直到物品出现）。
3. **激活态 (Active)**: 检测到物品存在。显示自定义名称，无图标，功能可用。

**代码实现**
- `src/角色浮岛/index.ts`:
  - `STORAGE_RECYCLE_ACTIVE_KEY`: 新增布尔值存储键，用于标记“是否已正式激活”。
  - `handleRecycleSave`: 仅保存名称和选项，**不再**直接修改 DOM 或播放解锁动画，也不设置 ACTIVE 状态。
  - `renderInventoryPanel`: 承担核心状态管理职责。
    - **检查**: 遍历物品列表查找自定义名称。
    - **状态流转**:
      - 找到物品 -> 设置 `ACTIVE = true` -> 渲染解锁状态。
      - 没找到物品 & `ACTIVE` 为真 -> 判定为物品丢失 -> 清除所有数据（重置） -> 渲染锁定状态。
      - 没找到物品 & `ACTIVE` 为假 -> 判定为等待生成 -> 保持数据但渲染锁定状态。

**自检确认**
- [x] 保存设置后，若物品未出现，按钮保持锁定状态。
- [x] 物品出现后，按钮自动刷新为解锁状态。
- [x] 物品消失后，按钮自动重置为初始锁定状态，且清除已保存的设置。

======================================

### 修复点击外部关闭逻辑冲突 [已完成] ✓

**问题描述**
- 用户反馈：
  1. 点击人物关系面板中的角色卡外部（空白处），会导致角色卡和人物关系面板同时关闭。
  2. **[新增]** 点击选项气泡外部（空白处），会导致选项气泡和浮岛（如果展开）同时关闭。
- 期望行为：
  1. 点击角色卡外部应只关闭角色卡；再次点击空白处才关闭人物关系面板。
  2. 点击选项气泡外部应只关闭选项气泡；再次点击空白处才收起浮岛。
- 原因分析：
  1. **事件冒泡/传播**：点击事件在关闭上层元素（角色卡/选项气泡）后继续传播，触发了底层的“点击外部关闭面板/收起浮岛”逻辑。
  2. **重复绑定**：全局点击事件处理器可能被多次绑定（特别是在热重载后），导致一次点击触发多次处理逻辑。

**修复方案**
- **阻止事件传播**：在 `globalClickHandler` 中，当成功处理并关闭了弹出卡片（`$popupCard`）或选项气泡（`state.isOptionsOpen`）时，显式调用 `e.stopImmediatePropagation()`，阻止事件继续传播给后续的处理器。
- **防止重复绑定**：在绑定全局 `click` 事件时，使用 `.off('click.namespace').on('click.namespace', ...)` 模式，确保同一命名空间的事件处理器只存在一个实例。
- **命名空间管理**：为不同面板的关闭逻辑分配独立的事件命名空间（如 `click.ci_global`, `click.ci_sub_close`, `click.ci_rel_bg` 等），便于精细化管理和调试。

**代码片段**
```typescript
// 1. 阻止事件传播 (选项气泡)
if (state.isOptionsOpen) {
  // ... 关闭气泡逻辑 ...
  state.isOptionsOpen = false;
  // [Fix] 关闭选项气泡后立即停止，防止同时触发收起浮岛逻辑
  if (e && e.stopPropagation) e.stopPropagation();
  return;
}

// 2. 阻止事件传播 (弹出卡片)
if ($popupCard.length) {
  // ...
  $popupCard.remove();
  if (e && e.stopPropagation) e.stopPropagation();
  // [Fix] 阻止后续事件处理器，确保只关闭卡片
  if (e && e.stopImmediatePropagation) e.stopImmediatePropagation();
  return;
}

// 3. 防止重复绑定
$(document).off('click.ci_global').on('click.ci_global', globalClickHandler);
```
