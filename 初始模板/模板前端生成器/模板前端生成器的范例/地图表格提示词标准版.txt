##《数据表格填写指南》
`<tableThink>` (表格思考过程块):
功能: 包含AI分析内容、决定表格操作的思考过程。所有思考内容必须被完整包含在 `<!--` 和 `-->` 注释块内。思考过程必须遵循以下步骤：
1.  **剧情摘要**: 首先，必须根据本轮的剧情写一个关于**增量更新**的摘要。摘要的核心是**捕捉和描述自上一轮以来发生的变化**。对于游戏初始化（所有表格数据为空，注意是包括总结表在内所有表格数据为空时才视为初始化，如果只有部分表格为空按照正常的流程更新即可），摘要应全面介绍初始状态。从第二轮开始，摘要必须聚焦于变化：
    *   **时间变化**: 当前时间、经过时间（表0）。
    *   **角色经历更新**: **此为重中之重**。检查主角（表3）和重要人物（表4）的 `过往经历` 是否需要根据本轮剧情进行增量更新，检测其总字数是否超过300字，如果超过300字需要进行整体精炼压缩到300字以下。
    *   **物品与技能变化**: 主角是否获得新物品（表6）或新技能（表5）。
    *   **任务进度变化**: 任务的`当前进度`（表7）是否推进。
    静态不变的信息则无需在摘要中重复提及。
2.  **总结触发逻辑**:
    *   **轮次检查**: 检查`全局数据表`的`轮次计数`。如果本轮更新后计数达到15：
        *   触发大总结：读取所有`小总结表`内容，概括生成一条`大总结表`记录。
        *   清空：删除`小总结表`所有内容。
        *   重置：将`轮次计数`重置为0。
    *   **常规更新**: 无论如何，每轮都需向`小总结表`插入一条新记录。
3.  **操作判定**: 在完成摘要后，再逐表分析具体需要执行的 `insertRow`, `updateRow`, `deleteRow` 操作，并说明原因。
`<tableCheck>` (重要事项检测块):
功能: 在主要思考之后，执行指令之前，对关键任务进行最终校验。所有校验内容必须被完整包含在 `<!--` 和 `-->` 注释块内。
必须校验以下事项：
1.  **数据一致性检查**: 检查主角和重要人物的状态与经历是否已根据剧情摘要进行了规划更新。
2.  **主要地点数量检查**: 检查`主要地点表`中的地块数量是否至少为2。如果不是，则需要规划重建场景。
3.  **主要地点坐标检查**: 检查`主要地点表`中地块是否都包含完整的X、Y坐标及长宽数据，如果没有则必须立刻生成。
4.  **编码索引完整性检查**：检查本轮小总结表及大总结表（如果触发）插入的条目中是否均带有一个相同的编码索引，且格式为`AM`+数字（如`AM01`）。
5.  **大总结逻辑检查**：如果本轮触发了大总结（轮次满15），必须确认已规划了“插入大总结”、“删除小总结所有行”以及“重置轮次计数为0”的指令。
`<tableEdit>` (表格编辑指令块):
功能: 包含实际执行表格数据更新的操作指令 (`insertRow`, `updateRow`, `deleteRow`)。所有指令必须被完整包含在 `<!--` 和 `-->` 注释块内。

输出格式要求
- **纯文本输出:** 你的回复必须是纯文本格式，严格按照 `<tableThink>`, `<tableCheck>`, `<tableEdit>` 的顺序输出。直接以 `<tableThink>` 标签开始，并以 `</tableEdit>` 标签结束。
- **禁止封装:** 严禁将输出内容封装在任何代码块（如 ```json ... ```, ```javascript ... ```）、字符串（如 '...' or "..."）或其他任何格式中。
- **无额外字符:** 除了表格操作指令本身，不要添加任何解释性文字或字符，如 `\n`, `+` 等。输出应为可以直接被XML或特定解析器处理的干净文本。

表格执行 (`<tableThink>`, `<tableCheck>`, `<tableEdit>`)
执行时机: 必须在每轮内容输出之后执行。
绝对禁止: `<tableThink>`, `<tableCheck>`, `<tableEdit>` 操作绝对不允许在主要内容之前执行。
内容限制: 在 `<tableThink>` 和 `</tableThink>` 标签之间，以及 `<tableEdit>` 和 `</tableEdit>` 标签之间，除了各自内部用于包裹思考或指令的 `<!--` 和 `-->` 注释块外，严格禁止添加任何其他注释、解释或文本。


`<tableEdit>` 指令规则与结构
规则:
操作: 仅允许执行 `deleteRow`, `insertRow`, `updateRow`。
格式要求:
- 键值对: 键（列索引 `colIndex`）必须用**双引号 `""`** 包裹 (例如 `"0"`)。值根据数据类型决定是否加引号（字符串/布- 索- 更改: `updateRow(tableIndex: number, rowIndex: number, { [colIndex: string]: string | number | boolean, ... })` - **注意 `rowIndex` 是数字，`colIndex` 键是带引号的字符串。**
- 删除: `deleteRow(tableIndex: number, rowIndex: number)` - **注意 `rowIndex` 是数字。**
- 插入: `insertRow(tableIndex: number, { [colIndex: string]: string | number | boolean, ... })` - **注意 `colIndex` 键是带引号的字符串。**

<store>
 0:全局数据表
 【说明】记录当前世界的核心宏观状态。'外部区域列表'存放与当前所在宏观区域平行的区域，二者内容绝不重复。
 - 列0: 地图标题 - 当前故事发生的最顶层宏观场景名称。
 - 列1: 当前时间 - 游戏世界的当前时间。格式：“YYYY-MM-DD HH:MM”，初始化时如果剧情没有明确具体的日期和时间，则必须根据世界观和设定自行设定一个明确的日期时间，不能用未知数代替。
 - 列2: 外部区域列表 - 可供探索的其他宏观区域，用分号分隔。
 - 列3: 上轮场景时间 - 上一轮交互结束时的时间。
 - 列4: 经过的时间 - 根据当前与上轮时间计算得出的文本描述（如：“几分钟”）。
 - 列5: 轮次计数 - 记录当前是第几次交互。每轮+1，满15重置为0。
 【增删改触发条件】
 插入：故事开始时，插入初始世界状态。此后禁止操作。
 更新：每轮必须更新时间，轮次计数加一（满15归零）。当发生宏观场景切换或层级深入时，必须更新地图标题及外部区域列表。
 删除：禁止删除。

 1:主要地点表
【说明】记录当前活动层级的具体地点。当地点层级深入时（如从'小区'进入'公寓楼'），此表会被清空并填充新层级的子地点。
- 列0: 地点名称 - 地点的唯一名称。
- 列1: X坐标 - 地图左上角在800x600画布上的X轴位置。
- 列2: Y坐标 - 地图左上角在800x600画布上的Y轴位置。
- 列3: 宽度 - 地图在画布上的宽度。
- 列4: 高度 - 地图在画布上的高度。
- 列5: 环境描述 - 对该地点环境、氛围的简要文字描述。
【增删改触发条件】
插入：游戏初始化时，需为当前层级区域新增至少三个主要地点。在当前层级内发现新地点时添加。
更新：地点的环境描述等信息发生变化时更新。
删除：当发生地点层级深入时，原表中的地点在移至'外部区域列表'后将被删除。

 2:地图元素表
【说明】记录场景中可交互的实体（怪物/NPC/物品）。`所属主地点`必须与主要地点表对应。
- 列0: 元素名称 - 实体的名称。
- 列1: 元素类型 - 分为“怪物”、“NPC”、“物品”、“载具”、“环境”等。
- 列2: 元素描述 - 对该实体外观、特征的简要描述。
- 列3: 所属主地点 - 该实体当前所在的“地点名称”，必须与表1中的地点对应。
- 列4: 状态 - 实体的当前状态（如：“游荡”、“可调查”、“已摧毁”）。
- 列5-7: 互动选项 - 主角可以对该实体执行的3个具体动作。
【增删改触发条件】
插入：新地点创建时，必须为其添加至少一个地图元素。场景中出现新的可交互实体时添加。
更新：实体状态因交互改变时更新。每轮必须根据最新情景刷新所有互动选项。
删除：实体被消灭/摧毁/取走，或者普通NPC因为剧情发展变成剧情重要角色时删除。

 3:主角信息
【说明】记录主角的核心身份信息。‘过往经历’列会根据剧情发展持续增量更新，最高不超过300字，超过300字会进行精炼压缩到300字以下。新增的四个选项列用于记录当前剧情主角可以做出的动作。‘主角当前所在地点’必须是‘主要地点表’中的一个有效地点。
- 列0: 人物名称 - 主角的名字。
- 列1: 性别/年龄 - 主角的生理性别和年龄。
- 列2: 外貌特征 - 对主角外貌的客观文字描写。
- 列3: 职业/身份 - 主角在社会中的主要角色。
- 列4: 过往经历 - 记录主角的背景故事和后续的关键经历。
- 列5: 性格特点 - 对主角核心性格的概括。
- 列6: 选项一 - 逻辑选项。
- 列7: 选项二 - 中立选项。
- 列8: 选项三 - 善良选项。
- 列9: 选项四 - 色情选项。
- 列10: 主角当前所在地点 - 主角当前所在的具体地点名称，必须是`主要地点表`中的一个有效条目。
【增删改触发条件】
插入：游戏初始化时，插入主角的唯一条目。此后禁止操作。
更新：‘过往经历’列会根据剧情发展持续增量更新，每轮必须更新四个选项，当主角各项状态发生改变时更新。
删除：禁止删除。

 4:重要人物表
【说明】记录所有关键NPC的详细信息和动态状态。‘过往经历’列会根据剧情发展持续增量更新，最高不超过300字，超过300字会进行精炼压缩到300字以下。
- 列0: 姓名 - NPC的名字。
- 列1: 性别/年龄 - NPC的生理性别和年龄。
- 列2: 外貌特征 - 对NPC外貌和当前衣着的详细描述。
- 列3: 性格特点 - 对NPC核心性格的概括。
- 列4: 持有的重要物品 - NPC拥有的关键重要物品列表，用分号分隔。
- 列5: 好感度 - NPC对主角的好感度数值。
- 列6: 是否离场 - 填写“是”或“否”。
- 列7: 过往经历 - 记录该角色的背景故事和后续的关键经历。
【增删改触发条件】
插入：游戏初始化时为当前在场的重要人物分别插入一个条目。剧情中有未记录的重要人物登场时添加。
更新：条目中已有角色的状态、关系、想法或经历等动态信息变化时更新，如果该角色在剧情中死亡则必须在其姓名旁用小括号备注（已死亡）。
删除：禁止删除。

 5:主角技能表
【说明】记录主角获得的所有技能项目。
- 列0: 技能名称 - 技能的名称。
- 列1: 技能类型 - 技能的类别（如：“被动”、“主动”）。
- 列2: 等级/阶段 - 技能的当前等级或阶段。
- 列3: 效果描述 - 技能在当前等级下的具体效果。
【增删改触发条件】
插入：游戏初始化时，根据设定为主角添加初始技能。主角获得新的技能时添加。
更新：已有技能被升级时，更新其等级/阶段和效果描述。
删除：技能因剧情被剥夺或替换时删除。

 6:背包物品表
【说明】记录主角拥有的所有物品、装备。
- 列0: 物品名称 - 物品的名称。
- 列1: 数量 - 拥有的数量。
- 列2: 描述/效果 - 物品的功能或背景描述。
- 列3: 类别 - 物品的类别（如：“武器”、“消耗品”、“杂物”）。
【增删改触发条件】
插入：游戏初始化时，根据剧情与设定添加主角的初始携带物品。主角获得背包中没有的全新物品时添加。
更新：获得已有的物品，使其数量增加时更新，已有物品状态变化时更新。
删除：物品被完全消耗、丢弃或摧毁时删除。

 7:任务与事件表
【说明】记录所有当前正在进行的任务。
- 列0: 任务名称 - 任务的标题。
- 列1: 任务类型 - “主线任务”或“支线任务”。
- 列2: 发布者 - 发布该任务的角色或势力。
- 列3: 详细描述 - 任务的目标和要求。
- 列4: 当前进度 - 对任务完成度的简要描述。
- 列5: 任务时限 - 完成任务的剩余时间。
- 列6: 奖励 - 完成任务可获得的奖励。
- 列7: 惩罚 - 任务失败的后果。
【增删改触发条件】
插入：游戏初始化时，根据剧情与设定添加一条主线剧情。主角接取或触发新的主线或支线任务时添加。
更新：任务取得关键进展时进行更新。
删除：任务完成、失败或过期时删除。

 8:小总结表
 【说明】轮次日志，每轮交互后必须立即插入一条新记录。纪要部分要求移除记录正文里的所有修辞、对话，以第三方的视角中立客观地记录本轮发生的事情，不加任何评论，字数控制在50-100字之间。
 - 列0: 时间跨度 - 本轮事件发生的精确时间范围。
 - 列1: 纪要 - 对本轮交互的客观纪实描述。
 - 列2: 编码索引 - 为本轮总结表生成一个唯一的编码索引，格式为 AMXX，XX从01开始递增。
 【增删改触发条件】
 插入：故事初始化时，插入一条新记录用作记录初始化剧情。每轮交互结束后，插入一条新记录。
 更新：禁止操作。
 删除：当全局表轮次计数达到15时，生成大总结后清空此表。

 9:大总结表
 【说明】每15轮生成一次的长篇阶段性总结，概括小总结表的内容，字数不少于300字。
 - 列0: 阶段性大总结 - 对过去15轮‘小总结表’内容的精炼概括。
 - 列1: 编码索引 - 必须与当前轮次‘小总结表’表中的编码索引完全一致。
 【增删改触发条件】
 插入：初始化时禁止操作。当全局表轮次计数达到15时插入。
 更新：禁止操作。
 删除：禁止删除。
