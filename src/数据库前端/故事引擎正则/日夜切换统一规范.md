# 日夜切换统一规范

## 结论

后续所有“日夜切换 / 昼夜切换”都统一使用当前 [`变量更新美化正则-日夜切换版`](变量更新美化正则-日夜切换版) 的交互方式：

- 不单独显示“日间 / 夜间”按钮。
- 不额外放置可见切换按钮。
- 复用标题栏左侧主图标作为切换入口。
- 日间图标显示 `✦`。
- 夜间图标显示 `✧`。
- 点击左侧图标区域切换日夜。
- 点击标题栏其他区域仍负责展开 / 收起。
- 夜间模式下左侧图标不加左侧描边，按夜间主题自身的图标背景和颜色走。
- 日夜模式必须持久化保存到 `localStorage`，刷新页面或新消息渲染后仍保持上次选择。
- 所有日夜切换面板共用统一存储键 `tavern_helper_day_night_theme`，任意面板切换后同步当前页面内其他日夜面板。

## 标准交互结构

以变量更新面板为标准：

```html
<details class="theme-state">
  <summary class="theme-button" aria-label="切换日夜模式"></summary>
</details>

<details class="shell">
  <summary class="panel-summary">
    <span class="toggle-icon">✦</span>
    <span class="summary-text">标题文字</span>
    <span class="fold-icon">▼</span>
  </summary>
</details>
```

关键点：

- `theme-state` 是不可见状态开关。
- `theme-button` 是透明点击热区。
- `theme-button` 覆盖在标题栏左侧 `toggle-icon` 的位置。
- `toggle-icon` 承担视觉显示。
- `theme-state[open] ~ .shell` 切换主题变量。
- `toggle-icon::before` 切换 `✦ / ✧`。

## 持久化脚本规范

### COAT 设计结论

- 关联：酒馆正则替换后的普通 `<script>` 片段可能不会稳定执行，导致只写了 `localStorage` 代码但实际没有完成持久化。
- 候选：继续使用逐面板 `<script>`、改成全局 `document.addEventListener('toggle')`、改成图片 `onload` 启动器 + `ontoggle` 单点处理。
- 评估：图片 `onload` 启动器在消息 HTML 注入场景中更可靠；`window.THDN` 单例只注册一次 `storage` 监听；主题点击用 `ontoggle` 就地触发，不再给每条消息重复绑定全局 `toggle` 监听。
- 回溯：废弃旧版 `window.__tavernHelperDayNightThemePersistReady` + 全局 `toggle` 监听方案，避免脚本不执行和多实例重复监听。
- 落盘：所有非完整 HTML 正则片段统一使用 `class="th-theme-init"` 的隐藏图片启动器；MVU 完整 HTML 前端使用同名 `window.THDN` 模块函数。

### 统一存储键

```text
tavern_helper_day_night_theme
```

取值：

```text
day
night
```

### 标准运行时对象

所有日夜面板共享 `window.THDN`：

```text
window.THDN.theme() 读取浏览器 localStorage 缓存
window.THDN.save(theme) 写入浏览器 localStorage 缓存
window.THDN.apply(theme, scope) 同步当前作用域及 MVU class
window.THDN.toggle(details) 处理用户点击后的主题切换
window.THDN.init(element) 初始化新渲染面板
```

### 标准 HTML 结构

```html
<details class="xxx-theme-state" ontoggle="window.THDN&&window.THDN.toggle(this)">
  <summary class="xxx-theme-button" aria-label="切换日夜模式"></summary>
</details>

<img class="th-theme-init" alt="" src="data:image/gif;base64,R0lGODlhAQABAAAAACw=" style="display:none!important;width:0;height:0;"
     onload="初始化或复用 window.THDN，并从 localStorage 恢复主题">
```

### 当前统一同步范围

```css
.vu-theme-state, .se-theme-state, .wlog-theme-state, .rel-theme-state, .bp-theme-state, .mvu-theme-state
```

后续新增日夜切换面板时，需要把对应的 `xxx-theme-state` 类加入统一列表。目前状态栏好感度、BP 系统、MVU 状态栏日夜版已分别加入 `.rel-theme-state`、`.bp-theme-state`、`.mvu-theme-state`。MVU 状态栏额外依赖 `.mvu-root.mvu-night` 与 `body.mvu-night-body`，因此 `window.THDN.apply()` 必须同步切换这两个 class。

### 内存与冗余控制

1. 不再使用每个面板都注册一次 `document.addEventListener('toggle')` 的方案。
2. 运行时只保留一个 `window.THDN` 单例。
3. 只注册一个 `storage` 监听，用于跨同页 / 同源页面同步。
4. 主题状态变更由当前 `<details>` 的 `ontoggle` 触发，不需要全局扫描点击事件。
5. `apply()` 使用短暂 `lock` 避免程序化同步 `[open]` 时回写造成重复触发。

## 多正则组是否能合成一个正则

### 不能强行全部合成一个正则

对于像故事引擎、BP 面板这种“复数正则组”，不建议强行合成一个正则。

原因：

1. 结构层级不同
   外层面板、事件条、NPC 卡、目标卡片通常来自不同原始文本片段。

2. 捕获目标不同
   外层正则负责捕获整体块；子正则负责逐条捕获列表项或子标签。强行合成会让查找正则非常长且脆弱。

3. 替换顺序依赖明确
   多条正则流水线更适合酒馆环境：先建外壳，再替换内部子项。

4. 维护成本更低
   子正则只负责结构，外层正则只负责主题和容器。后续改样式不需要重写所有捕获逻辑。

因此，最佳方案不是“所有东西合成一条正则”，而是：

```text
一组正则 = 一个外层主题宿主 + 多个结构子正则
```

## 多个正则如何统一切换昼夜模式

### 推荐方案：外层主题宿主

复数正则组统一采用：

```text
01_外层面板_日夜切换
02_子结构A_日夜共用
03_子结构B_日夜共用
04_子结构C_日夜共用
```

### 01 外层正则负责

- 输出共同根容器。
- 输出不可见的 `theme-state`。
- 输出标题栏左侧透明点击热区。
- 输出 `✦ / ✧` 图标切换 CSS。
- 输出日间 CSS 变量。
- 输出夜间 CSS 变量覆盖。
- 输出所有子卡片的共享样式。

### 02+ 子正则负责

- 只输出结构。
- 不输出 `<style>`。
- 不输出日夜按钮。
- 不写死日间 / 夜间颜色。
- 使用外层正则已经定义好的 class。

## 标准文件计划

### 1. 单条正则

适用对象：

- 变量更新面板
- 简单折叠说明块
- 单块正文面板

处理方式：

```text
一条正则内完成：
- 透明 theme-state
- 左侧图标点击热区
- ✦ / ✧ 图标切换
- 日夜 CSS 变量
- localStorage 持久化脚本
- 面板内容
```

当前示范文件：

- [`变量更新美化正则-日夜切换版`](变量更新美化正则-日夜切换版)

### 2. 复数正则组

适用对象：

- 故事引擎悬浮球
- BP 系统面板
- 多目标战斗卡片
- 多 NPC / 多事件面板

处理方式：

```text
第 1 条正则：
  外层主题宿主 + 标题栏图标切换 + 全部 CSS + localStorage 持久化脚本

后续正则：
  只输出子卡片结构
```

当前示范文件：

- [`故事引擎悬浮球美化正则-日夜切换版`](故事引擎悬浮球美化正则-日夜切换版)

## 后续落地计划

### 阶段 1：固定规范

- 已以变量更新面板确认最终交互：
  - 左侧图标点击切换。
  - `✦ / ✧` 图标变化。
  - 不显示额外按钮。
  - 使用 `localStorage` 保存主题状态。
- 本规范文件记录为后续标准。

### 阶段 2：修正故事引擎日夜版

需要把 [`故事引擎悬浮球美化正则-日夜切换版`](故事引擎悬浮球美化正则-日夜切换版) 的按钮方式改成同样规范：

- 不显示独立“日间 / 夜间”按钮。
- 复用悬浮球或标题栏左侧主图标区域作为切换入口。
- 日间 `✦`，夜间 `✧`。
- 子正则继续只输出结构。

### 阶段 3：处理 BP 系统日夜版

BP 系统已按复数正则组方式落地，不强行合成单正则：

```text
01_BP外层面板_日夜切换
02_BP目标卡片_日夜共用
03_BP其他子块_日夜共用
```

外层统一控制：

- 稀有度颜色变量
- 面板背景变量
- 条形图变量
- 标题图标 `✦ / ✧`
- 子卡片样式

### 阶段 4：处理状态栏

MVU 状态栏已按完整 HTML / 前端片段方式落地。为保持一致，继续按同样交互：

- 标题栏左侧主图标点击切换。
- `✦ / ✧` 随日夜变化。
- 不显示单独按钮。

### 夜间

- 显示名：夜间
- 视觉名：暗色
- 图标：`✧`

推荐：

```text

```

```text

```

旧文件可保留作为参考，但新文件中统一使用“暗色”命名。

## 复制使用原则

给用户看的正则文件必须清楚分区：

```text
## 你需要复制哪部分
## 替换内容
```

代码块里只放要复制的 HTML / CSS，不要在代码块内部写说明性注释。

说明文字放在代码块外。
