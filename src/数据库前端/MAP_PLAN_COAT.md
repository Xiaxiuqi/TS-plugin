# 角色浮岛地图功能升级规划 (COAT)

## 1. Context (背景)

当前“角色浮岛”插件的地图功能仅支持简单的 2D 矩形区域（`X, Y, W, H`），缺乏层级关系（如“一楼”、“二楼”），也无法表现复杂的室内结构（墙壁、家具、门窗）。用户希望实现类似《星穹铁道》的地图体验，包含：

- **层级地图**：支持世界 -> 区域 -> 楼层/房间的嵌套结构。
- **动态绘制**：根据描述自动生成墙壁、家具等视觉元素。
- **交互性**：玩家可以点击地图上的元素进行互动（如“调查”、“移动”）。

## 2. Objective (目标)

设计并实现一套全新的地图系统，核心包括：

1. **新型数据结构**：设计支持嵌套、矢量绘图和元数据的 JSON 结构，存储于数据库的“布局数据”列中。
2. **AI 提示词工程**：设计 System Prompt，使 AI 能从自然语言描述中提取出结构化数据。
3. **前端渲染引擎**：开发基于 SVG 或 Canvas 的渲染器，解析数据并绘制可交互的地图。

## 3. Action (行动方案)

### 3.1 数据结构设计 (Data Structure)

我们将不再依赖扁平的 CSV 表格列来存储所有信息，而是引入一个 `LayoutData` 字段（存储为 JSON 字符串），用于描述复杂的内部结构。

**层级模型：**
- **World (世界)**: 宏观区域，对应“全局数据表”中的 `外部区域列表`。
- **Area (区域)**: 当前所在的宏观地点（如“黑塔空间站”），对应“主要地点表”的一行。
- **MapNode (地图节点)**: 区域内的具体房间或地点（如“主控舱段”）。
- **Layer (层级)**: 房间内的分层（如“F1”, “F2”）。
- **Element (元素)**: 墙壁、家具、NPC、物品等。

**JSON Schema 草案 (`LayoutData`):**

```json
{
  "version": "1.0",
  "layers": [
    {
      "id": "F1",
      "name": "一层",
      "zIndex": 0,
      "bounds": { "x": 0, "y": 0, "w": 800, "h": 600 },
      "walls": [
        { "start": [0, 0], "end": [800, 0], "type": "solid" },
        { "start": [800, 0], "end": [800, 600], "type": "solid" },
        // ... 更多墙壁坐标
      ],
      "furniture": [
        { "id": "desk_01", "type": "desk", "x": 100, "y": 100, "w": 50, "h": 30, "rotation": 0 },
        { "id": "bed_01", "type": "bed", "x": 200, "y": 100, "w": 60, "h": 80, "rotation": 90 }
      ],
      "portals": [
        { "id": "door_to_f2", "targetLayer": "F2", "x": 750, "y": 300, "type": "stairs" }
      ]
    }
  ]
}
```

### 3.2 AI 提示词设计 (Prompt Engineering)

需要设计专门的 `Map Generation Prompt`，指导 AI 将环境描写转化为上述 JSON。

**输入示例：**
> “这是一个宽敞的大厅，大概20米见方。北面有一扇巨大的落地窗，东面是通往二楼的旋转楼梯。大厅中央放着一张圆形的地毯和几个沙发。西面墙上挂着一幅油画，下面有一个壁炉。”

**输出要求：**

1. 估算坐标系（假设 1米 = 10像素）。
2. 生成 `walls` 数据闭合房间。
3. 生成 `furniture` 数据（沙发、地毯、壁炉）。
4. 生成 `portals` 数据（楼梯）。

**提示词策略：**
- **CoT (思维链)**: 先让 AI 分析空间布局（方位、尺寸），再生成 JSON。
- **Few-Shot**: 提供 2-3 个标准的“描述 -> JSON”范例。
- **格式约束**: 强制输出纯 JSON，不包含 Markdown 代码块标记以外的文本。

### 3.3 前端渲染与交互 (Frontend)

- **渲染技术**: 使用 SVG (Scalable Vector Graphics)。SVG 天然支持交互事件绑定，且适合绘制矢量墙壁和图标。
- **组件化**:
  - `MapContainer`: 负责缩放、平移、层级切换。
  - `LayerRenderer`: 渲染特定层级的 SVG 内容。
  - `Element`: 具体的家具或 NPC，支持点击、悬停效果。
- **交互逻辑**:
  - **点击家具/物品**: 弹出详情卡（复用现有物品详情逻辑），显示描述和操作按钮（“检查”、“拿取”）。
  - **点击传送门**: 切换显示的 Layer。
  - **拖拽**: 平移地图视图。
  - **滚轮**: 缩放地图。

## 4. Test (测试计划)

1. **数据解析测试**: 手动构造复杂的 JSON 数据，验证前端能否正确渲染出墙壁、家具和层级切换。
2. **AI 生成测试**: 使用 GPT-4 或 Claude 测试提示词，评估生成的 JSON 是否符合 Schema，坐标是否合理（没有重叠、越界）。
3. **交互测试**: 模拟用户点击、拖拽，确保事件响应灵敏，无穿透或冒泡问题。
4. **性能测试**: 在包含大量（100+）元素的地图中测试渲染性能。
