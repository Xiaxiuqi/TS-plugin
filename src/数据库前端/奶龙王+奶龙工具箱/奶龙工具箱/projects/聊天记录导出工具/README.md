# 聊天记录导出工具 v6.0.8

JS-Slash-Runner（酒馆助手）「聊天记录导出」脚本的版本仓库。

## 目录约定

```
projects/聊天记录导出工具/
├── README.md                 本文件：当前版本 + 完整更新日志（唯一更新日志位置）
├── latest/                   最新版（仅一个版本，与 archive 中最高版本对应）
│   ├── 酒馆助手脚本-聊天记录导出 v<X>.json
│   └── 聊天记录导出工具 v<X>.js
└── archive/                  历史版本，每版一个目录
    ├── v3.7/酒馆助手脚本-聊天记录导出 v3.7.json
    ├── v3.8/...
    └── v<X>/...
```

> `latest/` 不再放更新日志。所有脚本本体更新日志都写在本文件。

## 当前版本

- 版本：v6.0.8
- JSON：`latest/酒馆助手脚本-聊天记录导出 v6.0.8.json`
- JS：`latest/聊天记录导出工具 v6.0.8.js`
- 历史版本：`archive/v3.7` ~ `archive/v6.0.7` 共 28 个。

## 修改本脚本的流程

每次修改本脚本前必须执行：

1. 把 `latest/` 当前脚本文件整体复制到 `archive/v<当前版本>/`。
2. 决定新版本号：
   - 新增小功能 / 兼容性修复 → 次版本 +0.1（如 v6.0 → v6.1）
   - 大量重构 / 行为不兼容 / 新模块 → 主版本 +1.0（如 v6.0 → v7.0）
   - 仅微调 / 修小 bug → 末位 +0.0.1（如 v6.0 → v6.0.1）
3. 在 `latest/` 中重命名 JSON / JS 为新版本号。
4. 修改脚本本体，并更新新 JSON 的 `info` 字段。
5. 更新本 README 的「当前版本」和「更新日志」。
6. 运行 `npm run validate`、`npm run check-refs`、`npm run test`。

详见仓库根 `docs/SCRIPT_VERSIONING.md` 与 `AGENTS.md`。

## 更新日志

> 仅记录脚本本体变更；项目级工程化变更见仓库根 `CHANGELOG.md`。

### v6.0.8

修复问题：

- 修复移动端点击楼层消息上的书签按钮无反应的问题。
- 楼层书签按钮补充 `touchend` / `pointerup` 触发，并对移动端合成 `click` 做去重，避免重复弹窗。
- 扩大楼层书签按钮触摸命中区，并让内部图标不拦截指针事件，提升移动端点击稳定性。

### v6.0.7

修复问题：

- 修复导航/搜索页滚动时外层容器的 `border-top`（上边框 / top border edge）跟随内容移动的问题。
- 参考文本导出页结构，将导航/搜索页改为外层 `#tab-nav` 固定、内部 `.cet-nav-scroll` 滚动。
- 保持 v6.0.6 已恢复的简单滚动模型，不重新引入移动端 media 覆盖或 JS 滚动守卫。

### v6.0.6

修复问题：

- 按 v5.2.4 的可用滚动实现回退滚动相关改动：移除移动端 media 覆盖、`touch-action` / `overscroll-behavior` 和 JS 触摸/滚轮守卫。
- 文本导出页恢复 `.cet-txt-scroll { overflow-y: scroll }`，导航/搜索和数据备份页恢复普通 `overflow-y: auto`。
- 保留 v6 系列新增功能，只回退影响移动端与桌面端滚动的布局/事件改动。

### v6.0.5

修复问题：

- 修复移动端文本导出页底部按钮区被主内容高度挤出面板，导致导出 TXT / EPUB 按钮不显示的问题。
- 保持电脑端“导出按钮贴在底部边框上”的设计：内容区滚动，按钮区固定在面板底部。
- 修复电脑端鼠标滚轮无法滚动内部内容、只能拖动滚动条的问题；为内部滚动容器补充 wheel 事件转发。

### v6.0.4

- 修复文本导出页下滑后回弹，底部导出按钮无法触达的问题。
