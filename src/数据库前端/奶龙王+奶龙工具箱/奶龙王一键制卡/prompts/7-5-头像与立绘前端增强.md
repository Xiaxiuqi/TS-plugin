# 头像与立绘前端增强提示词

## 这个文件负责什么

当用户希望在任意前端中显示以下内容时，使用本文件作为**可选增强步骤**：

1. 角色头像
2. 情绪差分头像
3. 立绘 / CG 图片
4. 随机立绘 / 随机 CG
5. 需要根据角色名、情绪、组名动态获取图片资源的前端效果

本文件**不是独立替代某个前端文件的主流程**，而是一个可插入到多个前端步骤中的**通用增强模块**。

它适用于但不限于：

- `frontend/opening-regex.html`
- `frontend/status-bar.html`
- `frontend/推进美化.html`
- 其他需要在 iframe 正则前端中展示头像、情绪头像、立绘、CG 的 HTML 前端文件

如果用户没有提出头像、立绘、CG 展示需求，则**不要强行启用本步骤**。

- 理解 `window.top.BubbleAvatar` 的调用方式
- 理解头像、情绪头像、CG、随机 CG 的不同用法
- 理解在 iframe 中必须等待 API 就绪
- 理解这是一套**前端辅助能力**，不是强制组件

### 禁止行为

1. ❌ 禁止在没有图片展示需求时强行加入头像 / 立绘逻辑
2. ❌ 禁止不读知识库就凭记忆乱写接口名
3. ❌ 禁止直接假设 API 一定已经就绪
4. ❌ 禁止把头像 / CG 写死成固定 URL，却还声称是动态接口方案
5. ❌ 禁止让可选增强反客为主，覆盖前端主流程本身

1. 本次是否真的需要头像 / 立绘 / CG 展示
2. 这些图片能力要加到哪个前端里
3. 每种图片的来源方式是什么
4. 是固定角色、固定情绪，还是动态切换
5. 最终把方案回填到对应前端文件，而不是孤立悬空

## 这是“可选流程”，不是“默认必做流程”

### 什么时候启用

当用户明确提出以下任一类需求时，才应启用本步骤：

- “前端里想显示角色头像”
- “状态栏里想显示当前人物头像 / 情绪头像”
- “开场白里想显示立绘 / CG”
- “推进面板里想显示角色头像或事件 CG”
- “希望用角色名、情绪、组名动态取图” 

### 什么时候不要启用

以下情况通常**不需要**启用本步骤：

- 用户只需要纯文本前端
- 用户只需要表格、按钮、变量显示
- 用户没有提出图片展示需求
- 当前前端结构还没确认，先做图片增强会干扰主流程

本步骤是**跨前端的辅助知识库步骤**，不会替代原有主步骤：

## 知识库中的接口能力

### 1. CG 图片接口

```javascript
const url = await window.top.BubbleCG.getImage('组名', 序号);
const randomUrl = await window.top.BubbleCG.getRandomImage('组名');
```

适合：

- 指定事件 CG
- 随机切换某一组立绘 / 场景图
- 开场白或推进前端中的大图展示

### 2. 角色头像接口

```javascript
const avatarUrl = await window.top.BubbleAvatar.getAvatar('角色名');
const moodAvatarUrl = await window.top.BubbleAvatar.getMoodAvatar('角色名', '情绪');
const color = await window.top.BubbleAvatar.getColor('角色名');
```

适合：

- 状态栏头像
- 对话面板头像
- 情绪状态切换头像
- 根据角色主题色做边框 / 高亮 / 标签色

---

## iframe 中的调用前提

### API 就绪等待

由于这些接口通过 `window.top` 暴露，在 iframe 中调用前必须先检查：

```javascript
function waitBubbleAPI(run) {
    if (!window.top || !window.top.BubbleAvatar || !window.top.BubbleCG) {
        setTimeout(() => waitBubbleAPI(run), 500);
        return;
    }
    run();
}
```

### 更稳妥的写法

如果本前端只用头像接口，则至少检查 `window.top.BubbleAvatar`。

如果本前端只用 CG 接口，则至少检查 `window.top.BubbleCG`。

不要因为一个接口不用，就强制等待另一个接口。

## 推荐使用场景

### A. 开场白前端

可用于：

- 在 `opening-regex.html` 中显示角色头像或立绘
- 在开局时根据固定角色名加载主头像
- 根据表单选择或剧情分支加载不同 CG

### B. 状态栏前端

可用于：

- 显示当前角色头像
- 根据状态栏中的“情绪 / 心情 / 态度”字段切换差分头像
- 使用角色主题色增强状态栏视觉层次

### C. 剧情推进前端

可用于：

- 根据推进标签显示角色头像
- 根据事件标签显示对应 CG
- 在推进结果面板中做角色卡片 / 事件卡片视觉增强

## 不同类型图片的接入原则

### 角色主头像

适合角色固定明确、情绪变化不重要的场景：

```javascript
const url = await window.top.BubbleAvatar.getAvatar('角色名');
```

### 情绪差分头像

适合有情绪变量、状态变量、标签切换的场景：

```javascript
const url = await window.top.BubbleAvatar.getMoodAvatar('角色名', '喜悦');
```

### 指定 CG

适合特定事件、固定场景：

```javascript
const url = await window.top.BubbleCG.getImage('组名', 1);
```

### 随机 CG

适合一组图轮换展示：

```javascript
const url = await window.top.BubbleCG.getRandomImage('组名');
```

---

## 动态变量接入原则

如果头像 / 立绘来源依赖当前前端里的变量，必须先确认这些变量来自哪里：

- 来自表格列
- 来自预设标签
- 来自固定配置
- 来自用户表单输入

### 禁止行为

- ❌ 禁止变量来源都没确认，就直接写动态接口逻辑
- ❌ 禁止状态栏缺列时，在前端里偷偷硬编码角色名冒充动态显示
- ❌ 禁止推进前端没有对应标签时，反过来要求预设配合某个临时图片方案
