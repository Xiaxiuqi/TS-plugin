# 组件规划总控提示词

## 这个文件只负责一件事

当世界书主体条目已经基本完成，准备进入组件阶段时，先使用本文件做**组件总控规划**。

本文件**不直接负责生成具体组件文件**，而是负责：

1. 询问用户这次到底需要哪些组件
2. 整理组件之间的依赖关系与制作顺序
3. 生成 / 更新带“前置阅读要求”的 todo list
4. 决定接下来应该进入哪个子步骤文档

具体组件制作必须分流到以下独立文档：

- 表格模板 → `prompts/6-表格模板讨论.md`
- 开场白前端 → `prompts/7-1-开场白前端.md`
- 状态栏前端 → `prompts/7-2-状态栏前端.md`
- 剧情推进预设 → `prompts/7-3-剧情推进预设.md`
- 剧情推进前端 → `prompts/7-4-剧情推进前端.md`
- 头像 / 立绘前端增强（可选） → `prompts/7-5-头像与立绘前端增强.md`

---

## ⚠️⚠️⚠️ 最高优先级规则：进入组件阶段前，必须先阅读本文件与范例规划 ⚠️⚠️⚠️

开始组件阶段前，必须先阅读：

1. 当前文档 `prompts/7-开场白状态栏讨论.md`
2. 范例项目规划文件 `examples/full-card/轮回禁则/current-plan.md`

目的：
- 理解一个完整项目通常包含哪些组件
- 理解组件不是一个大杂烩步骤，而是可拆成多个独立步骤
- 理解后续 todo list 应如何按步骤拆开

### 禁止行为

1. ❌ 禁止不做组件规划就直接开始写 `opening.html`
2. ❌ 禁止把开场白、状态栏、剧情推进预设、推进前端混成一个大步骤
3. ❌ 禁止没有 todo list 就开始做组件
4. ❌ 禁止 todo list 不写“开始前先读什么文档、什么范例”

---

## 阶段目标

与用户确认：

1. 本次组件阶段到底要做哪些东西
2. 哪些组件是必须的，哪些是不需要的
3. 各组件之间的依赖顺序
4. 每一步开始前必须阅读哪些文档和范例

---

## 先问什么：组件结构确认

进入本阶段时，必须先询问用户本次组件结构。

### 必问项

```text
在继续之前，我先确认这次组件阶段的范围：

1. 表格模板是否需要？
2. 开场白前端是否需要？（opening.html + opening-regex.html）
3. 状态栏前端是否需要？（status-bar.html）
4. 剧情推进预设是否需要？（剧情推进.json）
5. 剧情推进前端是否需要？（推进美化.html + 推进正则配置.json）
6. 是否需要头像 / 立绘前端增强？（角色头像、情绪头像、立绘 / CG）
7. 如果需要图片展示，是要放在哪个前端里？

如果有明确不需要的部分，也请直接说。
确认后，我会先生成一份 todolist，并在每一步前注明必须先阅读的文档、范例，以及图片知识库（如有需要）。
```

### 组件判断原则

- **表格模板**：只要状态栏、开场白初始化、剧情推进变量需要依赖表格，通常都需要先做
- **开场白前端**：只要需要开局展示、开始按钮、初始化表格 / 预设，就需要
- **状态栏前端**：只要需要持续显示数据，就需要
- **剧情推进预设**：只要用户需要“正文前额外推进 / 最终注入替换 / 额外DM裁定 / 规则判定辅助”之一，就需要
- **剧情推进前端**：只要启用了剧情推进预设，通常就应该配套制作，不能长期缺省
- **头像 / 立绘前端增强**：只有用户明确提出头像、情绪头像、立绘、CG 展示时才启用，属于附着在具体前端之上的可选增强步骤

---

## 组件依赖顺序

默认建议顺序如下：

1. 表格模板
2. 剧情推进预设（如需要）
3. 开场白前端（如需要）
4. 状态栏前端（如需要）
5. 剧情推进前端（如需要）
6. 打包确认

### 为什么这样排

- 状态栏前端依赖表格列结构，因此通常不能早于表格模板
- 开场白前端若要初始化表格或注入预设，需要知道表格与预设是否已经定稿
- 剧情推进前端必须建立在剧情推进预设已经定稿的前提上，否则会出现“展示层倒逼预设层”的错误

### 强制依赖规则

1. 如果状态栏要读取某些字段，但表格里没有这些列：
   - 必须先回到表格模板步骤补列
   - 禁止在状态栏里硬编码兜底

2. 如果开场白需要在开始按钮中一并注入剧情推进预设：
   - 必须先完成剧情推进预设步骤
   - 再做开场白前端

3. 如果剧情推进预设尚未定稿：
   - 禁止先做推进前端
   - 禁止让推进美化层反向规定 recall 规则

## todo list 生成规则（本阶段核心）

组件结构一旦确认，就必须立即生成或更新 todo list。

### todo 必须满足的要求

1. 每一项都必须是一个明确步骤
2. 每一项都必须写清楚：
   - 开始该步前要先阅读哪个提示词文档
   - 开始该步前要先阅读哪个范例文件
3. 要按依赖顺序排列
4. 用户中途增删组件时，必须同步更新

### 推荐 todo 写法

```markdown
[ ] 阅读 docs/建表指导/自定义表建表指南.md 与 docs/变量语法/syntax-reference.md 与 prompts/6-表格模板讨论.md 与 examples/full-card/轮回禁则/tables/table-template.json，再确认表格模板结构与变量
[ ] 阅读 prompts/7-3-剧情推进预设.md 与 examples/full-card/轮回禁则/frontend/剧情推进.json，再确认剧情推进预设结构与 recall 规则
[ ] 阅读 prompts/7-1-开场白前端.md 与 examples/full-card/轮回禁则/frontend/opening.html、examples/full-card/轮回禁则/frontend/opening-regex.html，再确认开场白前端方案
[ ] 阅读 prompts/7-2-状态栏前端.md 与 examples/full-card/轮回禁则/frontend/status-bar.html，再确认状态栏数据来源与展示方案
[ ] 阅读 prompts/7-4-剧情推进前端.md 与 examples/full-card/轮回禁则/frontend/推进美化.html、examples/full-card/轮回禁则/frontend/推进正则配置.json，再确认剧情推进前端方案
[ ] 阅读 prompts/7-5-头像与立绘前端增强.md 与 docs/图片与前端知识库/气泡格式对话图片调用知识库.md，再确认是否需要头像 / 立绘 / CG 增强以及接口调用方式
[ ] 阅读 prompts/8-打包确认.md 与 examples/full-card/轮回禁则/current-plan.md、examples/full-card/轮回禁则/轮回禁则.json，再进行打包前总审查
```

### todo 生成后的执行纪律

- 进入某一步前，先完成该步写明的阅读动作
- 没有完成阅读，不能开始该步讨论、草稿、写入
- 任何新发现的依赖，也要追加进 todo list

---

## 本文件的正确输出

本文件阶段的输出应该只有三类：

1. **组件需求确认结果**
2. **组件执行顺序**
3. **更新后的 todo list**

本文件阶段**不应直接输出**：
- 完整 `opening.html`
- 完整 `opening-regex.html`
- 完整 `status-bar.html`
- 完整 `剧情推进.json`
- 完整 `推进美化.html`
- 完整 `推进正则配置.json`

这些都应在各自子文档阶段完成。

## 组件规划确认模板

```text
让我确认一下这次组件阶段的规划：

【需要的组件】
- 表格模板：[需要 / 不需要]
- 开场白前端：[需要 / 不需要]
- 状态栏前端：[需要 / 不需要]
- 剧情推进预设：[需要 / 不需要]
- 剧情推进前端：[需要 / 不需要]
- 头像 / 立绘前端增强：[需要 / 不需要]
- 图片增强落点：[开场白前端 / 状态栏前端 / 剧情推进前端 / 其他前端 / 无]

【建议顺序】
1. [步骤1]
2. [步骤2]
3. [步骤3]
...

【执行规则】
- 每一步开始前，必须先阅读对应提示词文档
- 有范例的步骤，必须先阅读对应范例文件
- 我会先更新 todolist，再进入下一步

请确认这个组件规划是否正确。
```

## 下一步分流规则

规划确认后，按以下规则进入下一步：

### 情况 A：需要表格模板

先进入：
- `prompts/6-表格模板讨论.md`

### 情况 B：不需要表格，但需要剧情推进预设

先进入：
- `prompts/7-3-剧情推进预设.md`

### 情况 C：只需要开场白前端

先进入：
- `prompts/7-1-开场白前端.md`

### 情况 D：只需要状态栏前端

先进入：
- `prompts/7-2-状态栏前端.md`

### 情况 E：剧情推进预设已定稿，准备做推进前端

进入：
- `prompts/7-4-剧情推进前端.md`

### 情况 F：某个前端还需要头像 / 立绘 / CG 展示增强

在对应前端主步骤确认后，额外读取：
- `prompts/7-5-头像与立绘前端增强.md`
- `docs/图片与前端知识库/气泡格式对话图片调用知识库.md`

## 一句话总结

**本文件只负责“先规划、再拆步、先列 todo、每步先阅读、然后再分流到子文档执行”，绝不把所有组件混成一个步骤。**
