# 需求分析提示词

## 阶段目标

通过对话明确用户的创作需求，包括：
- 创作类型
- 主题方向
- 风格基调
- 特殊需求

## ⚠️ 重要规则：确认后再推进

**每个阶段结束前，必须获得用户明确确认！**

- 确认后才记录到Plan文件
- 确认后才进入下一阶段
- 用户沉默或模糊回答时，继续追问或等待

## ⚠️⚠️⚠️ 本阶段新增最高优先级任务：先确认流程目的与组件结构，再生成 todo list

在需求分析阶段，除了确认创作类型、主题、风格外，还必须额外确认：

1. **本次流程的目的**
   - 完整制作一张新卡
   - 只做世界书主体
   - 只补某些组件

2. **本次大致结构**
   - 是否需要表格模板
   - 是否需要开场白前端
   - 是否需要状态栏前端
   - 是否需要剧情推进预设
   - 是否需要剧情推进前端（`推进美化.html` + `推进正则配置.json`）
   - 是否需要头像 / 立绘前端增强（角色头像、情绪头像、立绘 / CG）

3. **确认后必须立刻生成或更新 todo list**
   - todo 必须覆盖本次预计执行的步骤
   - 每一项都要写明：开始该步前必须阅读哪个提示词文档
   - 若该步有范例资产，还要写明：开始该步前必须阅读哪个范例文件
   - 若该步涉及头像 / 立绘 / CG 展示，还要写明：开始该步前必须阅读 `prompts/7-5-头像与立绘前端增强.md` 与 `docs/图片与前端知识库/气泡格式对话图片调用知识库.md`

## 分析框架

### 创作类型判断

通过提问确定用户想要创作的内容类型：

```
请选择您想创作的内容类型：

1. **世界观设定**
   - 完整的世界背景设定
   - 适合开放世界、探索向剧情
   - 包含规则、势力、地点等

2. **人物角色卡**
   - 以角色为核心的创作
   - 适合角色扮演、对话向剧情
   - 包含人物设定、背景故事等

3. **剧情向角色卡**
   - 以故事线为核心的创作
   - 适合线性/分支剧情
   - 包含剧情节点、转折等

请告诉我您的选择，或者描述您的想法，我来帮您判断。
```

### 主题方向询问

确定类型后，询问主题方向：

```
很好，您选择了[世界观设定]。

请描述一下您心中的创意或主题：

- 有没有初步的故事背景或设定？
- 想要什么样的氛围？（轻松/黑暗/悬疑/浪漫...）
- 有参考的作品或风格吗？

可以用一两句话简单描述，或者告诉我"不确定"，我会通过提问帮您梳理。
```

### 规模判断（世界观类型时）

如果用户选择世界观，进行规模预判：

```
关于世界观的规模，您有想法吗？

1. **微型场景** - 单一地点，如一座建筑、一个设施
   - 适合：密室逃脱、恐怖场景、特殊场所

2. **中型区域** - 一个城市或组织
   - 适合：城市探险、势力斗争、组织内部

3. **宏大世界** - 国家、星球、多元宇宙
   - 适合：史诗冒险、文明冲突、宏大叙事

如果您不确定，我可以根据您的主题建议合适的规模。
```

### 风格基调确认

```
最后，关于风格基调：

1. **现实向** - 贴近现实，注重逻辑
2. **奇幻/科幻向** - 包含超自然或科技元素
3. **混合向** - 现实+奇幻/科幻元素

您倾向哪种风格？
```

### 组件结构预规划（新增必问）

```
在正式进入后续步骤前，我还需要先确认本次流程的大致结构：

1. 本次目标更接近哪一种？
   - 完整新卡
   - 主体设定 + 少量组件
   - 只补组件

2. 你预计需要哪些组件？
   - 表格模板
   - 开场白前端（opening.html + opening-regex.html）
   - 状态栏前端（status-bar.html）
   - 剧情推进预设（剧情推进.json）
   - 剧情推进前端（推进美化.html + 推进正则配置.json）
   - 头像 / 立绘前端增强（角色头像、情绪头像、立绘 / CG）

3. 有没有明确不需要的组件？

4. 如果需要图片展示，计划放在哪个前端里？
   - 开场白前端
   - 状态栏前端
   - 剧情推进前端
   - 其他前端

确认后，我会先生成一份 todolist，并在每一步前标注“必须先阅读的文档、范例，以及图片知识库（如有需要）”。
```

## 需求记录格式

完成需求分析后，在 `projects/<卡名>/current-plan.md` 中记录：

```markdown
## 需求分析结果

### 基本信息
- **创作类型**：世界观设定 / 人物角色卡 / 剧情向角色卡
- **主题方向**：
- **规模**（世界观时）：micro / mid / macro
- **风格基调**：现实向 / 奇幻科幻向 / 混合向

### 流程目的与组件结构
- **本次流程目的**：完整新卡 / 主体设定 + 少量组件 / 只补组件
- **表格模板**：需要 / 不需要
- **开场白前端**：需要 / 不需要
- **状态栏前端**：需要 / 不需要
- **剧情推进预设**：需要 / 不需要
- **剧情推进前端**：需要 / 不需要
- **头像 / 立绘前端增强**：需要 / 不需要
- **图片增强落点**：开场白前端 / 状态栏前端 / 剧情推进前端 / 其他前端 / 无

### 用户原始描述
> [用户的原始描述内容]

### 特殊需求
- [如有特殊需求列出]

### todo 生成要求
- 已根据当前结构生成或更新 todo list
- todo 中每一步均已写明前置阅读文档
- 有范例的步骤已写明前置阅读范例
- 涉及图片展示的步骤已写明前置阅读 `prompts/7-5-头像与立绘前端增强.md` 与 `docs/图片与前端知识库/气泡格式对话图片调用知识库.md`

### 阶段状态
- [x] 需求分析完成
- [ ] 进入规模判断/框架确定阶段
```

## 阶段结束确认

完成需求分析后：

```
让我确认一下您的需求：

**创作类型**：[类型]
**主题方向**：[主题]
**规模**：[规模]
**风格基调**：[风格]
**本次流程目的**：[完整新卡 / 主体设定 + 少量组件 / 只补组件]
**组件结构**：[表格模板 / 开场白前端 / 状态栏前端 / 剧情推进预设 / 剧情推进前端 / 头像与立绘前端增强]
**图片增强落点**：[开场白前端 / 状态栏前端 / 剧情推进前端 / 其他前端 / 无]

是否正确？如有需要调整的地方请告诉我。

确认后，我会先生成或更新一份带“前置阅读要求”的 todo list，再进入下一阶段：[规模判断/框架确定]。
```

需求确认后：

2. 世界观设定 → 读取 `2-世界观规模判断.md`
3. 人物角色卡 → 读取 `4-人物设计.md`
4. 剧情向角色卡 → 读取 `5-剧情设计.md`
