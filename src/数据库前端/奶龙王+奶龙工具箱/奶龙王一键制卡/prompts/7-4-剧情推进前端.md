# 剧情推进前端提示词

## 这个文件负责什么

当组件规划已经确认“需要剧情推进前端”时，使用本文件。

本文件只负责：

1. `frontend/推进美化.html`
2. `frontend/推进正则配置.json`

如果剧情推进预设本体还没有定稿，不要使用本文件，先去：
- `prompts/7-3-剧情推进预设.md`

如果还没有确认组件结构，先回到：
- `prompts/7-开场白状态栏讨论.md`

---

## ⚠️⚠️⚠️ 最高优先级规则：开始前必须先阅读本文件与范例 ⚠️⚠️⚠️

在讨论或生成剧情推进前端之前，必须先阅读：

1. `prompts/7-4-剧情推进前端.md`
2. `examples/full-card/轮回禁则/frontend/推进美化.html`
3. `examples/full-card/轮回禁则/frontend/推进正则配置.json`
4. 已定稿的 `frontend/剧情推进.json`

如果用户希望在推进前端中显示角色头像、情绪头像、事件 CG 或随机 CG，还必须额外阅读：

5. `prompts/7-5-头像与立绘前端增强.md`
6. `docs/图片与前端知识库/气泡格式对话图片调用知识库.md`

### 阅读目标

- 学习推进美化 HTML 的展示职责
- 学习推进正则配置的 `beautify` / `cleanup` 结构
- 学习如何围绕既有 `extractTags` 设计展示
- 学习为什么展示层不能反向定义 recall 规则
- 如需图片展示，学习如何通过知识库接口接入头像、情绪头像、立绘与 CG

### 禁止行为

1. ❌ 禁止在预设本体未定稿前先做推进前端
2. ❌ 禁止推进美化层自己补 recall 内容
3. ❌ 禁止让 `推进正则配置.json` 与实际注入头部格式不匹配
4. ❌ 禁止缺少 `推进美化.html` 或 `推进正则配置.json` 其中之一

## 阶段目标

与用户确认并完成：

1. 推进结果打算如何展示
2. 实际会消费哪些 `extractTags`
3. `beautify` 与 `cleanup` 如何配置
4. 最终形成 `推进美化.html` 与 `推进正则配置.json`

---

## 推进前端的定位

### `推进美化.html`

- 这是展示层
- 负责把预设已输出的标签结果美化显示
- 它不是剧情推进预设本体
- 它不是完整 regex script 成品，而是后续打包转换来源

### `推进正则配置.json`

- 用于定义两段配置：
  - `beautify`
  - `cleanup`
- 需要与当前卡的最终注入文本头部格式一致
- 不能依赖项目统一默认配置兜底

---

## 与剧情推进预设本体的依赖关系

### 强制前提

开始做推进前端前，必须已经明确：

1. 当前卡的 `frontend/剧情推进.json` 已定稿
2. `extractTags` 已定稿
3. recall 规则已定稿
4. `<recall>` 为空时的语义已定稿

### 原则

- 推进前端只消费结果
- 推进前端不定义规则
- 推进前端不放宽规则
- 推进前端不修补预设错误

---

## `推进美化.html` 的要求

### 强制规则

1. 必须是完整 HTML 文档
2. 必须是当前卡专属文件
3. 风格要与本卡的开场白前端、状态栏前端协调
4. 只能展示预设已经正确产出的标签

### 禁止行为

- ❌ 不要把它做成项目默认通用版直接复用
- ❌ 不要展示预设实际不会输出的标签
- ❌ 不要在这里重新定义 recall 的计算或来源

### 设计重点

- 围绕 `extractTags` 做面板拆分
- 把真正重要的标签信息分成可读区块
- 对空值、缺值做展示层处理，但不能捏造数据

---

## `推进正则配置.json` 的要求

### 必须包含两段

```json
{
  "beautify": { ... },
  "cleanup": { ... }
}
```

### `beautify`

- 用于生成剧情推进输入美化正则
- 要与 `推进美化.html` 配套

### `cleanup`

- 用于生成剧情推进高楼层隐藏正则
- 匹配锚点要与最终注入文本头部一致

### 禁止行为

- ❌ 缺任何一段
- ❌ 锚点与实际注入文本不一致
- ❌ 继续沿用别的卡的旧配置不改

## 必须向用户确认什么

### 推进前端确认模板

```text
在制作剧情推进前端前，我先确认以下内容：

【展示目标】
- 你希望推进结果重点展示哪些标签？
- 哪些标签只要简单展示，哪些要重点做视觉区分？

【标签来源】
- 当前预设已经定稿的 extractTags 是哪些？
- 是否只围绕这些标签展示？（正确答案通常是“是”）

【图片增强】
- 是否需要在推进前端中显示角色头像、情绪头像、事件 CG 或随机 CG？
- 如果需要，图片来源是固定值、标签值，还是其他动态变量？

【清理需求】
- 是否需要高楼层隐藏？
- 最终注入文本的头部锚点是否已经确定？

确认后，我会基于范例结构起草 `推进美化.html` 与 `推进正则配置.json`。
```

## 展示层只能消费标签，不能篡改标签

### 正确做法

- 标签有值 → 正常展示
- 标签为空 → 显示为空或“无”
- recall 为空 → 显示为空，不要补造

### 错误做法

- 因为界面太空，就自己补一段 recall 内容
- 因为某标签没出来，就在展示层硬凑文本
- 为了画面完整，偷偷放宽预设规则

---

## 风格一致性要求

推进前端是卡专属资产，因此必须与当前卡的：

- `opening-regex.html`
- `status-bar.html`

保持大体风格协调，例如：
- 颜色体系
- 卡片形态
- 标题层级
- 信息密度

但风格一致不代表复制粘贴，要根据推进标签结构调整展示逻辑。

---

## 打包约束

只要存在 `frontend/剧情推进.json`，且用户确认启用剧情推进链路，则通常应满足：

- `frontend/推进美化.html` 存在
- `frontend/推进正则配置.json` 存在

### 成套规则

- 三者应视为成套资产
- 缺一即为半成品
- 打包阶段应直接报错，不做静默跳过

## 一句话总结

**剧情推进前端只能围绕已定稿的 [`frontend/剧情推进.json`](examples/full-card/轮回禁则/frontend/剧情推进.json) 来做 [`推进美化.html`](examples/full-card/轮回禁则/frontend/推进美化.html) 和 [`推进正则配置.json`](examples/full-card/轮回禁则/frontend/推进正则配置.json)，绝不能让展示层倒过来规定预设层。**
