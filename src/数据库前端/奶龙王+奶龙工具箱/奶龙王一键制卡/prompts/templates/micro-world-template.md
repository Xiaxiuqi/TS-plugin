# Micro - 微观场景模板

## 适用范围

单一场所或小型封闭环境，如：
- 废弃医院、鬼屋
- 特定型号飞船、潜艇
- 神秘商店、特殊设施
- 封闭研究基地
- 异空间碎片

## 核心条目结构

### uid: 001 - 元数据

```yaml
uid: 001
comment: 场所元数据
content: |
  location_name: [场所名称]
  location_type: [场所类型]
  theme_tags: 场所 | [主题1] | [主题2] | [主题3]
  tone: [基调1] | [基调2]
position: before_char
enabled: true
depth: 4
```

### uid: 002 - 核心规则

```yaml
uid: 002
comment: 场所核心规则
content: |
  # [场所名称]的运作法则
  
  ## 进入条件
  [如何进入此场所]
  
  ## 离开条件
  [如何离开此场所]
  
  ## 行为规范
  [在此场所内的行为限制]
  
  ## 环境法则
  [特殊的物理/环境规则]
  
  ## 主要危险
  [场所内的主要威胁]
position: before_char
enabled: true
depth: 4
```

### uid: 003 - 环境概览

```yaml
uid: 003
comment: 场所环境概览
content: |
  # [场所名称]的环境布局
  
  ## 整体布局
  [整体结构描述：线性/迷宫/中心辐射/多层]
  
  ## 主导感官
  [视觉/听觉/嗅觉/触觉/味觉体验]
  
  ## 区域划分
  [主要区域A(别名): 特征, has(关键设施), 氛围描述]
  [主要区域B(别名): 特征, has(关键设施), 氛围描述]
  [次要区域C(别名): 特征, has(物品点), 氛围描述]
  [特殊区域D(别名): 特征, has(特殊现象), 氛围描述]
position: before_char
enabled: true
depth: 4
```

### uid: 004 - 特殊机制

```yaml
uid: 004
comment: 场所特殊机制
content: |
  # [场所名称]的独特机制
  
  ## 核心机制名称

  [如何触发此机制]

  [如何利用或避免此机制]

```

## PList格式示例

```
[急诊科(急救区): 24小时运营, 快节奏, has(抢救室, 创伤处理区, 分诊台), 专业应急团队, 配备先进急救设备]
[内科(诊疗区): 综合诊疗, 慢性病管理, has(诊室, 病房, 心电图室), 细致全面, 以患者为中心]
[外科(手术区): 手术治疗, 精密操作, has(手术室, 麻醉室, 恢复室), 高技术要求, 严格消毒]
[ICU(重症区): 重症监护, 生命支持, has(监护仪, 呼吸机, 除颤仪), 限制进入, 最高级别护理]
```

## 条目数量建议

- 核心条目：4-5个
- 区域条目：3-8个
- NPC条目：0-5个
- 物品条目：0-5个
- 事件条目：0-3个

**总计**：5-20个条目
