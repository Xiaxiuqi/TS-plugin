# YAML输出格式规范

## 文件命名

- 格式：`uid-xxx.yaml`
- 示例：`001-核心规则.yaml`, `015-人物设定.yaml`

## 基本结构

```yaml
uid: [三位数字]
comment: [条目标题]
content: |
  [条目内容，使用Markdown格式]
position: [插入位置]
enabled: [是否启用]
constant: [是否持续触发]
prevent_recursion: [是否禁止递归]
insertion_order: [插入顺序]
depth: [扫描深度]
role: [角色类型]
keys: [触发关键词列表]
secondary_keys: [次要触发关键词]
```

## 字段说明

### uid

- 类型：整数
- 格式：三位数字，从001开始
- 用途：唯一标识符

### comment

- 类型：字符串
- 用途：条目标题/备注
- 建议：简洁明了，便于识别

### content

- 类型：多行字符串
- 格式：使用 `|` 保留换行
- 内容：使用Markdown格式组织

### position

可选值：
- `before_char` - 角色定义之前（推荐：核心规则、世界观、势力、**地点**、物品）
  - 对应 extensions.position: 0
- `after_char` - 角色定义之后（推荐：**人物索引、人物详细设定**、特殊输出格式、动态模板）
  - 对应 extensions.position: 1
- `before_em` - 示例消息之前
  - 对应 extensions.position: 5
- `after_em` - 示例消息之后
  - 对应 extensions.position: 6
- `before_an` - 作者注释之前
  - 对应 extensions.position: 2
- `after_an` - 作者注释之后
  - 对应 extensions.position: 3
- `at_depth` - 特定深度
  - 对应 extensions.position: 4

#### ⚠️ 重要规则：人物相关条目放在角色定义之后

**所有人物相关的条目（人物索引、人物详细设定）都应该使用 `position: after_char`，放在角色定义之后。**

**地点相关条目（地点索引、地点详细设定）应该使用 `position: before_char`，放在角色定义之前。**

**原因**：
- 人物设定是在用户定义角色之后才需要加载的内容，不应该影响角色定义本身
- 地点是环境背景，应该在角色定义之前就存在

#### ⚠️ 重要规则：系统深度（depth: 0）的使用限制

**系统深度（depth: 0）只能用于特别特别重要的规则，且放置前需要向用户确认。**

**原因**：系统深度会干扰上下文，只有极重要的条目才需要使用。

**适用场景**：
- 必须始终生效的输出格式规则
- 必须覆盖其他所有设定的核心规则
- 极其重要的系统级指令

**注意**：普通条目不要使用depth: 0，会过度干扰上下文。

### depth

- 类型：整数
- 默认：4
- 用途：扫描深度

### enabled

- 类型：布尔值
- 默认：true
- 用途：控制条目是否激活

### constant

- 类型：布尔值
- 默认：true
- 用途：是否持续触发（始终存在于上下文）
- 蓝灯条目：`true`
- 绿灯条目：`false`

### prevent_recursion

- 类型：布尔值
- 默认：true
- 用途：是否禁止进一步递归
- 所有条目默认禁止递归

### insertion_order

- 类型：整数
- 默认：100
- 用途：控制同位置条目的插入顺序
- 数值越小，越先插入

#### ⚠️ 重要规则：提前规划条目顺序

**条目顺序和位置最好在框架确定阶段就规划好，有需要再修改。**

**推荐使用紧凑编号**，方便后续插入新条目：

| 条目类型 | uid范围 | insertion_order范围 | 说明 |
|---------|--------|-------------------|------|
| 核心条目 | 001-009 | 1-9 | 元数据、核心规则、剧情脉络 |
| 索引条目 | 004-009 | 4-9 | 人物索引、地点索引、物品索引 |
| 人物详细 | 010-099 | 10-99 | 每个人物一个uid |
| 地点详细 | 100-199 | 100-199 | 每个地点一个uid |
| 物品详细 | 200-299 | 200-299 | 每个物品一个uid |
| 其他条目 | 300+ | 300+ | 特殊规则、输出格式等 |

**示例规划**：
```
001 - 元数据 (insertion_order: 1)
002 - 核心规则 (insertion_order: 2)
003 - 剧情脉络 (insertion_order: 3)
004 - 人物索引 (insertion_order: 4)
005 - 地点索引 (insertion_order: 5)
010 - 主角详细 (insertion_order: 10)
011 - 妻子详细 (insertion_order: 11)
012 - 女儿详细 (insertion_order: 12)
013 - 仇敌详细 (insertion_order: 13)
100 - 豪宅详细 (insertion_order: 100)
```

**备用范围（仅供参考，不推荐）**：
- 核心世界观：1-10
- 核心规则：10-30
- 势力/组织概览：30-50
- 登场人物简介：50-70
- 城市/地点简介：70-90
- 重要物品简介：90-100
- 人物详细设定：100-200
- 地点详细设定：200-300
- 复杂规则详述：300-400
- 特殊输出格式：0-10（after_char）

### depth

- 类型：整数
- 默认：4
- 用途：扫描深度（position为at_depth时使用）

### role

- 类型：整数
- 默认：0
- 用途：角色类型
  - 0: 系统
  - 1: 用户
  - 2: 助手

### keys

- 类型：字符串数组
- 默认：[]
- 用途：触发关键词
- 绿灯条目必须设置关键词
- 蓝灯条目通常为空数组

### secondary_keys

- 类型：字符串数组
- 默认：[]
- 用途：次要触发关键词

## 蓝绿灯配置规则

### ⚠️ 重要规则：所有条目必须禁止递归

**所有条目都必须设置 `prevent_recursion: true`**，防止条目之间相互触发导致上下文膨胀。

### 蓝灯条目（始终存在）

**配置**：
```yaml
constant: true
prevent_recursion: true
keys: []
```

**特点**：
- 始终存在于上下文中
- 禁止递归触发其他条目
- 无需触发词

**适用条目**：
- 核心世界观
- 核心规则
- 势力/组织概览
- **人物索引**（所有人物名称和简介）
- **地点索引**（所有地点名称和简介）
- **物品索引**（所有物品名称和简介）
- 重要物品简介

### 绿灯条目（关键词触发）

**配置**：
```yaml
constant: false
prevent_recursion: true
keys: ["关键词1", "关键词2"]
```

**特点**：
- 当对话中出现关键词时触发
- 禁止递归触发其他条目
- 必须设置触发关键词

**适用条目**：
- **人物详细设定**（触发关键词为人物名）
- **地点详细设定**（触发关键词为地点名）
- **物品详细设定**（触发关键词为物品名）
- 复杂规则详述
- 特殊输出格式

## 索引条目+详细条目结构

### 规则说明

**人物、地点、物品必须采用"索引条目+详细条目"的双层结构**：

1. **蓝灯索引条目**：包含所有人物/地点/物品的名称和简要介绍
2. **绿灯详细条目**：每个人物/地点/物品的详细设定，通过关键词触发

### 索引条目示例（人物索引）

```yaml
uid: 004
comment: 人物索引
content: |
  # 人物索引
  
  ## 主要人物
  - **林远**：主角，32岁，入狱前是公司经理
  - **苏婉**：主角妻子，30岁，原家庭主妇
  - **林小雪**：主角女儿，12岁
  - **赵天成**：仇敌，45岁，商界大鳄
position: after_char  # ⚠️ 人物相关条目使用 after_char
enabled: true
constant: true
prevent_recursion: true
insertion_order: 50
depth: 4
role: 0
keys: []
```

### 详细条目示例（人物详细设定）

```yaml
uid: 010
comment: 主角-林远
content: |
  # 林远 - 主角详细设定
  
  ## 基本信息
  - 年龄：32岁
  - 身份：入狱前是公司经理
  - 性格：温和善良
  
  ## 背景故事
  [详细背景...]
position: after_char  # ⚠️ 人物相关条目使用 after_char
enabled: true
constant: false
prevent_recursion: true
insertion_order: 100
depth: 4
role: 0
keys: ["林远"]
```

### 索引条目示例（地点索引）

```yaml
uid: 005
comment: 地点索引
content: |
  # 地点索引
  
  ## 主要地点
  - **林家别墅**：故事主要发生地，原为林远的家
  - **公司**：林远曾经工作的地方
position: before_char  # ⚠️ 地点相关条目使用 before_char
enabled: true
constant: true
prevent_recursion: true
insertion_order: 5
depth: 4
role: 0
keys: []
```

### 详细条目示例（地点详细设定）

```yaml
uid: 100
comment: 地点-林家别墅
content: |
  # 林家别墅 - 详细设定
  
  ## 基本信息
  - 名称：林家别墅
  - 位置：城市郊区的高档住宅区
  
  ## 环境描述
  [详细描述...]
position: before_char  # ⚠️ 地点相关条目使用 before_char
enabled: true
constant: false
prevent_recursion: true
insertion_order: 100
depth: 4
role: 0
keys: ["别墅", "林家别墅", "家"]
```

### 为什么采用双层结构？

1. **节省上下文**：索引条目始终存在但内容精简，详细条目按需触发
2. **提高响应速度**：AI始终知道有哪些人物/地点/物品存在
3. **避免信息过载**：详细设定只在相关话题出现时才加载

## 内容格式规范

### 标题层级

```markdown
# 一级标题 - 用于条目主标题
## 二级标题 - 用于主要章节
### 三级标题 - 用于子章节
```

### 列表格式

```markdown
- 无序列表项1
- 无序列表项2

1. 有序列表项1
2. 有序列表项2
```

### 强调格式

```markdown
**粗体** - 用于强调关键信息
*斜体* - 用于补充说明
`代码` - 用于专有名词
```

### 链接与引用

```markdown
[链接文字](目标)
> 引用内容
```

## PList格式说明

用于环境描述和区域概览：

```
[名称(别名): 特征1, 特征2, has(设施1, 设施2), 氛围描述]
```

**示例**：
```
[急诊科: 24小时运营, 快节奏, has(抢救室, 创伤处理区), 专业团队]
[行政区: 政府核心, has(市政厅, 法院), 防御森严]
```

**元素说明**：
- 名称：区域/地点名称
- 别名：可选的替代名称
- 特征：用逗号分隔的特征描述
- has()：包含的设施或元素
- 最后部分：氛围或附加描述

## 完整示例

### 蓝灯条目示例（核心规则）

```yaml
uid: 001
comment: 核心规则
content: |
  # 世界核心法则
  
  ## 基本规则
  这个世界遵循以下基本法则：
  - 魔法存在于空气中，但需要天赋才能感知
  - 时间流速在不同区域可能不同
  
  ## 限制与约束
  - 凡人无法直接使用高级魔法
  - 某些区域被神明封印，无法进入
position: before_char
enabled: true
constant: true
prevent_recursion: true
insertion_order: 10
depth: 4
role: 0
keys: []
```

### 绿灯条目示例（人物详细设定）

```yaml
uid: 010
comment: 王城诺顿详情
content: |
  # 王城诺顿详情
  
  **核心区域**：
  巨石王宫坐落于内城高地，是王室与最高议政厅所在；圣光大教堂位于王宫东侧。
  
  **可能遭遇的重要事件**：
  1. **王室继承暗流**：奥德里克三世国王病重，王子与派系贵族围绕继承权展开明争暗斗。
  2. **圣光教会审判异端**：教会宣布发现城内潜伏崇拜"暗影"的异端分子。
  
  **可能碰到的重要人物**：
  - **奥德里克三世国王**：诺顿王国现任君主，年迈但威严尚存。
  - **大主教伊格纳修斯**：圣光教会在诺顿的最高领袖。
position: before_char
enabled: true
constant: false
prevent_recursion: true
insertion_order: 110
depth: 4
role: 0
keys: ["王城", "王室", "诺顿"]
```

## 注意事项

1. **内容缩进**：content字段内的内容从行首开始，不额外缩进
2. **特殊字符**：如需使用冒号、引号等，注意YAML语法
3. **编码**：使用UTF-8编码
4. **换行**：使用Unix换行符(LF)或Windows换行符(CRLF)均可
5. **蓝绿灯选择**：
   - 如果条目需要始终存在（如核心世界观、规则、简介），使用蓝灯
   - 如果条目只在特定话题出现时才需要（如详细设定），使用绿灯
6. **关键词设置**：
   - 绿灯条目必须设置关键词
   - 关键词应选择条目中最可能被提及的词汇
7. **插入顺序**：
   - 同类条目使用相近的 insertion_order
   - 越重要的条目 insertion_order 越小
