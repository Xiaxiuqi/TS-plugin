## 阶段目标

与用户确认并完成：

1. 状态栏是否真的需要
2. 需要显示哪些数据
3. 这些数据分别来自哪些表格、哪些列
4. 表格模板是否已具备这些列
5. 最终形成 `status-bar.html`

## 状态栏的数据来源规则

### 唯一正确原则

**状态栏所有数据必须从表格读取。**

状态栏数据流程：

```text
1. 状态栏加载时调用 API 读取表格
2. 解析表格内容
3. 提取需要显示的字段
4. 动态生成 HTML
5. 表格更新时重新渲染
```

### 绝对禁止的做法

- ❌ 在 HTML 里写死角色名
- ❌ 在脚本里写死属性数值
- ❌ 表格缺列时直接在前端里臆造字段
- ❌ 不检查列名就直接读数据

## 表格缺失时的处理流程

如果状态栏需要显示某项数据，但表格中没有对应列：

```text
步骤1：指出缺的是哪一列
    ↓
步骤2：说明应该加到哪张表里
    ↓
步骤3：等待用户确认是否修改表格模板
    ↓
步骤4：先回到表格模板步骤补列
    ↓
步骤5：再继续做状态栏
```

### 强制规则

- 表格不齐时，不能强行继续做状态栏定稿
- 状态栏不能成为“偷偷补数据”的地方

## `status-bar.html` 的格式要求

### 强制规则

1. 必须是完整 HTML 文档
2. 必须包含样式、容器、脚本
3. 必须动态渲染内容
4. 必须注册表格更新回调

### 正确骨架

```html
<!DOCTYPE html>
<html lang="zh-CN">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>状态栏</title>
</head>
<body>
  <div id="statusContainer"></div>
  <script>
    // 从表格读取数据并渲染
  </script>
</body>
</html>
```

## API 调用要求

### 获取 API 对象

必须优先尝试父窗口中的 `AutoCardUpdaterAPI`：

```javascript
function getAutoCardAPI() {
    if (typeof parent !== 'undefined' && parent.AutoCardUpdaterAPI) {
        return parent.AutoCardUpdaterAPI;
    }
    if (window.AutoCardUpdaterAPI) {
        return window.AutoCardUpdaterAPI;
    }
    return null;
}
```

### 注册更新回调

必须在初始化时注册：

```javascript
document.addEventListener('DOMContentLoaded', async function() {
    await loadTableData();
    const api = getAutoCardAPI();
    if (api) {
        api.registerTableUpdateCallback(loadTableData);
    }
});
```

### 读取表格数据

推荐通过 `exportTableAsJson()` 读取：

```javascript
async function loadTableData() {
    const api = getAutoCardAPI();
    if (!api) return;

    const jsonData = await api.exportTableAsJson();
    const tables = typeof jsonData === 'string' ? JSON.parse(jsonData) : jsonData;
    parseTables(tables);
    renderUI();
}
```

## 解析表格时的硬规则

### 基本方法

1. 先遍历所有表
2. 读取 `sheet.name`
3. 读取 `content[0]` 作为表头
4. 用列名查索引，不要写死下标

### 推荐辅助函数

```javascript
const get = (row, colName) => {
    const idx = headers.indexOf(colName);
    return idx !== -1 && idx < row.length ? row[idx] : undefined;
};
```

### 为什么不能写死下标

因为表格列顺序可能调整，状态栏必须尽量按列名读取，减少耦合。

## 显示风格确认

### 可选风格

1. **简洁型**
   - 适合信息少、更新频繁的状态
2. **卡片型**
   - 适合角色详情、多个对象并列展示
3. **表格型**
   - 适合严格结构化数据

### 选择原则

- 数据少且强调概览 → 简洁型
- 多角色 / 多对象展示 → 卡片型
- 大量规则化字段 → 表格型

---

## 多状态区支持

如果用户需要多个维度：

- 全局状态
- 主角状态
- 角色列表
- 物品或资源

可以放在同一个 `status-bar.html` 中拆分多个区块，但仍然属于一个前端文件。

### 注意

- 这是多个区块，不是多个独立状态栏文件
- 每个区块的数据来源都必须能追溯到具体表格和列

1. 显示什么内容
2. 从哪张表读取

4. 哪些列已存在
5. 哪些列仍缺失

## 一句话总结

**状态栏前端的核心不是“好看”，而是严格从表格读取、严格按列名解析、严格动态渲染；表格没准备好，就先别做状态栏定稿。**
