# 打包确认提示词

## 阶段目标

审阅所有创作内容，确认打包选项，生成最终角色卡JSON文件。

## ⚠️ 重要规则：确认后才打包

**在用户明确确认前，绝对禁止执行打包操作！**

- 展示所有内容让用户审阅
- 用户明确说"确认打包"/"可以打包"后，才执行打包命令
- 用户沉默或模糊回答时，继续追问

## ⚠️⚠️⚠️ 打包前必须先阅读本文件、范例，并核对 todo list ⚠️⚠️⚠️

在进入打包确认前，必须先阅读：

1. `prompts/8-打包确认.md`
2. `examples/full-card/轮回禁则/current-plan.md`
3. `examples/full-card/轮回禁则/轮回禁则.json`

并且必须先核对：

- todo list 中与组件有关的步骤是否都已完成
- 每个组件步骤是否都按规则先读过对应文档与范例
- 当前项目是否仍有半成品组件

**禁止行为：**

- ❌ 没完成 todo 就直接打包
- ❌ 组件步骤还没读文档和范例就直接打包
- ❌ 明知剧情推进资产缺件还继续打包

## 审阅清单

在展示审阅清单前，必须先补做一个检查：

```text
【前置检查】
- 本次流程目的是否已经明确？
- todo list 是否已生成并更新到最新？
- 每一步前置阅读是否已执行？
- 各组件是否都按拆分步骤完成，而不是混做？
- 如启用了头像 / 立绘前端增强，对应知识库与前端步骤是否已读完？
```

## 执行打包

**只有在“前置检查 + 审阅确认 + 组件完整性检查”都通过后，才能执行本节。**

```
正在打包角色卡...

1. 读取世界书条目...
   - 加载 X 个YAML文件

2. 读取数据库组件...
   - 加载表格模板
   - 加载开场白HTML
   - 加载状态栏HTML
   - 如启用：加载剧情推进预设
   - 如启用：加载推进美化HTML与推进正则配置
   - **先检查剧情推进预设本体的 recall 规则是否正确，再看美化层是否正常消费标签**

3. 编码正则脚本...
   - 开场白 → Base64编码
   - 状态栏 → 正则替换字符串
   - 如启用：剧情推进输入美化 → HTML转正则
   - 如启用：剧情推进高楼层隐藏 → 配置转正则
   - **注意：推进美化层不能兜底补 recall 内容；若预设应为空，则展示层也必须显示为空**

4. 生成角色卡JSON...
   - 合并所有组件
   - 生成完整结构

✓ 打包完成！

输出文件：output/[名称].json
```

## 异常处理

### 条目文件缺失

```
⚠️ 警告：以下条目文件缺失：
- uid-xxx.yaml

这些条目将被跳过。是否继续打包？
```

### 组件文件缺失

```
⚠️ 警告：以下组件文件缺失：
- opening.html

该组件将不会包含在角色卡中。是否继续？
```

### 剧情推进资产缺失

```
⚠️ 错误：检测到已启用剧情推进预设，但缺少配套文件：
- frontend/推进美化.html
- frontend/推进正则配置.json

剧情推进预设、推进美化HTML、推进正则配置必须成套出现。
请先补齐资产，再执行打包。
```
