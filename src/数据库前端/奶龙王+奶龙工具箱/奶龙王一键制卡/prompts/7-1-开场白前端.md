# 开场白前端提示词

## 这个文件负责什么

当组件规划已经确认“需要开场白前端”时，使用本文件。

本文件只负责以下两个文件：

1. `frontend/opening.html`
2. `frontend/opening-regex.html`

如果还没有确认是否需要开场白前端，先回到：
- `prompts/7-开场白状态栏讨论.md`

如果要做的是状态栏，不要使用本文件，改去：
- `prompts/7-2-状态栏前端.md`

如果要做的是剧情推进预设，不要使用本文件，改去：
- `prompts/7-3-剧情推进预设.md`

如果要做的是剧情推进前端，不要使用本文件，改去：
- `prompts/7-4-剧情推进前端.md`

- 学习 `opening.html` 的纯文本职责
- 学习 `opening-regex.html` 的完整 HTML 结构
- 学习 `$1` 占位符的用法
- 学习 `{{TEMPLATE_BASE64}}` 与 `{{PRESET_BASE64}}` 占位符的保留方式
- 学习开始按钮、初始化脚本、发送消息到酒馆的正确写法
- 如果用户需要在开场白中显示头像、情绪头像、立绘或 CG，还必须额外阅读 `prompts/7-5-头像与立绘前端增强.md` 与 `docs/图片与前端知识库/气泡格式对话图片调用知识库.md`

### 禁止行为

1. ❌ 禁止不读范例就直接写开场白前端
2. ❌ 禁止把开场白纯文本和开场白美化 HTML 混成一个文件
3. ❌ 禁止遗漏 `$1` 占位符
4. ❌ 禁止把表格数据或预设数据硬编码进 HTML
5. ❌ 禁止点击开始按钮后只注入数据、不发送消息到酒馆

## 阶段目标

与用户确认并完成：

1. 开场白要展示什么内容
2. 是否需要“开始游戏”按钮
3. 是否需要注入表格模板
4. 是否需要一并注入剧情推进预设
5. 最终形成 `opening.html` 与 `opening-regex.html`

## 开场白的两个文件不是同一个东西

### `opening.html`

- 作用：保存开场白纯文本内容
- 去向：打包进角色卡 `first_mes`
- 形式：纯文本内容，最终必须以 `<start>` 与 `</start>` 包裹

### `opening-regex.html`

- 作用：开场白美化正则 HTML
- 去向：作为正则脚本资源
- 形式：完整 HTML 文档，负责美化显示、开始按钮、初始化逻辑

### 强制理解

- `opening.html` 是内容层
- `opening-regex.html` 是展示层 + 初始化层
- 两者缺一不可（只要启用了开场白前端）

## `opening.html` 的格式要求

### 强制规则

1. 必须是开场白内容本体
2. 最终必须以 `<start>` 与 `</start>` 包裹
3. 不要把整页 HTML 写进这个文件
4. 不要把脚本写进这个文件

### 正确示例

```html
<start>
这里是开场白内容。
描述故事初始情境、人物状态、玩家切入点。
</start>
```

### 错误示例

```html
<html>
  <body>这是错误的，因为把 opening-regex.html 的职责混进来了</body>
</html>
```

---

## `opening-regex.html` 的格式要求

### 强制规则

1. 必须是完整 HTML 文档
2. 必须包含 `$1` 占位符用于展示开场白内容
3. 必须预留 `{{TEMPLATE_BASE64}}` 占位符
4. 如果支持剧情推进预设，必须预留 `{{PRESET_BASE64}}` 占位符
5. 必须有开始按钮和初始化脚本
6. 开始按钮成功后必须调用 `sendToTavern()`

### 必备结构

```html
<!DOCTYPE html>
<html lang="zh-CN">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
</head>
<body>
  <div id="openingContent">$1</div>
  <button id="startButton">开始游戏</button>

  <script>
    const EMBEDDED_TEMPLATE_BASE64 = "{{TEMPLATE_BASE64}}";
    const EMBEDDED_PRESET_BASE64 = "{{PRESET_BASE64}}";
  </script>
</body>
</html>
```

## 数据注入规则

### 表格模板注入

如果当前卡需要表格模板，`opening-regex.html` 必须：

```javascript
const EMBEDDED_TEMPLATE_BASE64 = "{{TEMPLATE_BASE64}}";
```

### 剧情推进预设注入

如果当前卡需要剧情推进预设，`opening-regex.html` 还必须：

```javascript
const EMBEDDED_PRESET_BASE64 = "{{PRESET_BASE64}}";
```

### 占位符规则

- 打包前必须保留占位符原文
- 不允许手动填入真实 Base64
- 不允许删掉占位符再期待打包脚本自动恢复

---

## 解码与初始化规则

### 解码函数要求

必须使用 UTF-8 解码 Base64 字符串，并安全处理异常。

### 正确初始化方式

必须通过 `window.parent.AutoCardUpdaterAPI.initGameSession({}, initOptions)` 初始化。

### 参数规则

- `injectTemplate: true` 表示注入表格模板
- `loadPreset: false` 表示不加载预设
- 如果存在预设，则改为 `loadPreset: true` 并传入 `presetData`
- 第一个参数是角色数据对象，可以为空对象 `{}`
- 第二个参数才是配置对象

### 禁止写法

```javascript
await api.initGameSession(tables);
```

这是错误的，因为把模板数据当成了第一个参数。

---

## 开始按钮点击后的强制行为

### 绝对禁止

1. ❌ 禁止只修改按钮文字，不做任何实际初始化
2. ❌ 禁止只注入表格，不发送消息到酒馆
3. ❌ 禁止假设用户会手动输入“开始”

### 正确流程

```text
1. 按钮变成“初始化中...”
2. 解码嵌入的模板 / 预设
3. 调用 initGameSession 注入表格，必要时一并注入预设
4. 注入成功后调用 sendToTavern()
5. 普通开场白发送“开始”
6. 表单类开场白发送表单内容
7. 按钮更新为“已开始”
```

---

## `sendToTavern()` 强制要求

必须使用直接操作酒馆 DOM 的方式，示意如下：

```javascript
function sendToTavern(message) {
    try {
        if (parent.document.getElementById('send_textarea')) {
            const textarea = parent.document.getElementById('send_textarea');
            const sendButton = parent.document.getElementById('send_but');

            if (textarea && sendButton) {
                textarea.value = message;
                sendButton.click();
                console.log('[游戏初始化] 消息已发送到酒馆');
            }
        }
    } catch (e) {
        console.error('[游戏初始化] 发送消息失败:', e);
    }
}
```

### 禁止方式

- ❌ 不要依赖 `window.parent.send`
- ❌ 不要只用 `postMessage` 指望酒馆自己接
- ❌ 不要省略消息发送逻辑

---

## 表单类开场白

如果用户要做“角色选择 / 属性选择 / 身份选择”类开场白：

1. 仍然要先完成表格 / 预设初始化
2. 然后收集表单字段
3. 把用户选择拼成文本
4. 使用 `sendToTavern()` 发送到酒馆

这类情况不能偷懒写成普通“开始”。

---

## 与剧情推进链路的关系

### 如果没有启用剧情推进预设

开场白前端仍然必须正常工作：
- 支持仅表格模板模式
- `loadPreset` 应保持为 `false`

### 如果启用了剧情推进预设

开场白前端必须：
- 支持同时注入表格模板与预设
- 不能要求用户额外再手动导入预设
- 但也不能替剧情推进预设本体补逻辑

### 如果还启用了头像 / 立绘增强

开场白前端可以额外接入：
- 角色主头像
- 情绪差分头像
- 指定 CG
- 随机 CG

但前提是：
- 必须先额外阅读 `prompts/7-5-头像与立绘前端增强.md`
- 必须先额外阅读 `docs/图片与前端知识库/气泡格式对话图片调用知识库.md`
- 必须明确图片来源是固定值、表单值还是其他动态变量
- 不能因为要加图片就破坏原有开始按钮、初始化与发送消息流程
