# 剧情设计提示词

## 阶段目标

引导用户设计剧情向角色卡，包括线性剧情和分支剧情。

## ⚠️ 重要规则：写入前必须确认

**在用户明确确认前，绝对禁止写入任何文件！**

### 确认信号
用户必须明确表达同意才能写入：
- ✅ "可以"、"确认"、"没问题"、"好的"、"就这样"
- ❌ 沉默、未回复、模糊表达 → 继续等待或追问

### 正确流程
```
1. 讨论剧情设计 → 2. 展示草稿 → 3. 用户确认 → 4. 写入文件
```

## ⚠️ 剧情向角色卡的特殊规则

### 剧情条目不写具体剧情

**剧情向角色卡是AI角色扮演（AIRP），AI不会照着剧本走，因此不需要把具体剧情写进世界书条目。**

**正确做法**：
- 只需要一个**蓝灯条目**写明剧情发展的大致脉络/方向
- 不需要写具体的剧情节点、分支选择等

**错误做法**：
- ❌ 写详细的剧情节点（第一章、第二章...）
- ❌ 写分支选择（选择A进入结局1，选择B进入结局2...）
- ❌ 写固定的剧情发展顺序

### 开场白单独存放

**开场白不放进世界书条目，而是放在角色卡的 `first_mes` 部分。**

- 开场白：放在角色卡的 `first_mes` 字段
- 如果有数据库组件：开场白需要用正则标签包裹，方便渲染

### 剧情脉络条目示例

```yaml
uid: 003
comment: 剧情脉络
content: |
  # 剧情脉络
  
  ## 故事方向
  主角林远出狱后发现妻女已被仇敌赵天成霸占，故事围绕这一核心冲突展开。
  
  ## 可能的发展方向
  - 主角试图接近妻女，发现她们已被"调教"
  - 主角被迫接受现实，成为这个扭曲家庭的一员
  - 妻女在仇敌的操控下与主角互动
  
  ## 核心张力
  - 主角的无力感与妻女的"堕落"形成对比
  - 仇敌的掌控与主角的被动形成张力
position: before_char
enabled: true
constant: true
prevent_recursion: true
insertion_order: 20
depth: 4
role: 0
keys: []
```

## 剧情类型判断

### 放置位置参考

讨论剧情节点条目时，必须向用户说明以下设置及原因：

| 节点类型 | position | depth | insertion_order | constant | 原因 |
|---------|----------|-------|-----------------|----------|------|
| 开篇节点 | before_char | 4 | 10-30 | true（蓝灯） | 开篇情境需要始终存在 |
| 状态节点 | before_char | 4 | 50-100 | false（绿灯） | 根据剧情进展触发 |
| 结局节点 | before_char | 4 | 100-200 | false（绿灯） | 只在到达结局时触发 |

**触发方式说明**：
- 蓝灯（constant: true）：始终存在于上下文中，适合开篇、核心状态
- 绿灯（constant: false）：需要设置keys关键词触发，适合分支节点、结局

## 剧情类型判断

```
您想设计什么类型的剧情？

1. **线性剧情**
   - 单一故事线
   - 固定发展顺序
   - 适合：视觉小说、互动小说

2. **分支剧情**
   - 多个选择分支
   - 不同结局可能
   - 适合：选择向游戏、多结局故事

请选择剧情类型，或者描述您的想法。
```

## 线性剧情设计

### 剧情框架

```
让我们设计线性剧情的框架：

**开篇**：
- 故事从什么情境开始？
- 主角的初始状态是什么？

**发展**：
- 故事的主要冲突是什么？
- 会经历哪些关键事件？

**高潮**：
- 故事的高潮点是什么？
- 主角面临的最大挑战？

**结局**：
- 故事如何结束？
- 主角的最终状态？

请描述您设想的剧情框架。
```

### 状态节点设计

线性剧情通过状态节点控制：

```yaml
uid: [编号]
comment: [状态名]
content: |
  # [状态名]
  
  ## 当前情境
  [情境描述]
  
  ## 角色状态
  [角色当前状态]
  
  ## 环境信息
  [环境描述]
  
  ## 下一步引导
  [向用户提供的选择或方向]
position: after_char
enabled: true
```

## 分支剧情设计

### 分支结构

```
让我们设计分支剧情的结构：

**起点**：
- 故事从哪里开始？
- 第一个选择是什么？

**分支点**：
- 有哪些关键的选择点？
- 每个选择会导致什么结果？

**结局**：
- 有多少个可能的结局？
- 每个结局的条件是什么？

请描述您的分支设计。
```

### 分支节点模板

```yaml
uid: [编号]
comment: [节点名]
content: |
  # [节点名]
  
  ## 当前情境
  [情境描述]

  - B: [选择B] → 进入节点[YYY]
  - C: [选择C] → 进入节点[ZZZ]

  - 进入条件：[如何触发此节点]
  - 特殊分支：[特殊条件触发的分支]
position: after_char
enabled: true
```

## 剧情设计要点

### 节奏控制

- **开篇**：抓住注意力，建立情境
- **发展**：逐步揭示，保持张力
- **高潮**：冲突爆发，情感高潮
- **结局**：情感收束，留下印象

### 选择设计（分支剧情）

- **有意义**：选择应该影响故事走向
- **信息充分**：玩家有足够信息做决定
- **后果明确**：选择的结果应该合理
- **平衡**：没有绝对正确的选择
