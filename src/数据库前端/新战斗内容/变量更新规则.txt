变量更新规则:
  MVU通用标准:
    check:
      - 所有变量更新必须写入 stat_data 下的既有字段路径，路径名称必须与变量结构完全一致
      - 只能更新剧情中确实发生变化的字段，禁止为了凑格式重复写入未变化字段
      - 动态字典字段使用对象键值对形式更新，不得写成数组
      - 数组字段必须使用数组形式更新，不得写成对象或字符串
      - 不得写入 "待初始化"，空集合统一使用 [] 或 {}
      - 数值字段只能写入数字，不得写入带单位的字符串
      - 布尔字段只能写入 true 或 false
      - 派生字段由变量结构自动计算时，除非规则明确允许，否则不直接修改
      - 删除动态对象中的条目时，必须删除对应键，而不是把值改成空字符串、0 或 false

  系统:
    时间:
      type: |-
        {
          年: number,
          月日: string,
          时分: string,
          星期: string
        }
      check:
        - 场景推进、行为耗时、休息、训练、任务执行后更新时分
        - 跨越24点时同步更新月日和星期
        - 时间必须保持连续性，不得无理由回退
        - 允许只更新发生变化的子字段，如只更新 时分

    地点:
      type: |-
        {
          国家: string,
          地域: string,
          场所: string,
          具体位置: string
        }
      check:
        - 仅在角色发生物理移动、转场、任务场景切换后更新
        - 允许只更新部分层级，如只改变 具体位置
        - 当前位置描述应保持由大到小的层级一致性

    性爱状态:
      type: |-
        {
          进行中: boolean,
          参与者: string[]
        }
      check:
        - 发生明确性行为或亲密行为进入持续状态时，将 进行中 设为 true
        - 参与者必须是角色名字符串数组，如 ["五条悟", "夏油杰"]
        - 行为结束后将 进行中 设为 false，并将 参与者 清空为 []
        - 禁止将 参与者 写成对象、布尔表或逗号分隔字符串

  user:
    等级:
      type: number
      check:
        - 禁止直接修改
        - 等级由 EXP 自动结算；每满100 EXP自动扣除100、等级+1、KP+5

    战力评级:
      type: string
      check:
        - 禁止直接修改
        - 由变量结构按 总BP 自动映射为 四级/三级/二级/准一级/一级/特级/强特级/论外

    EXP:
      type: number
      check:
        - 战斗结束、重大剧情节点、训练完成后增加
        - 同级战斗通常 +50，越级战斗 +200~500，碾压或无实质挑战不给经验
        - EXP 可超过100，变量结构会自动升级并保留溢出值
        - 禁止写入负数

    KP:
      type: number
      check:
        - 每升1级由变量结构自动获得5点 KP
        - 领悟技能、强化能力、修改束缚等明确消耗时可以减少
        - 禁止无剧情依据凭空增加
        - 禁止写入负数

    身体状况:
      type: string
      check:
        - 受伤、中毒、疲劳、恢复、治疗、脑损伤后更新
        - 描述应简洁，如 健康、轻伤、重伤、濒死、疲惫、中毒、脑损伤

    永久损伤或疤痕:
      type: string
      check:
        - 仅在产生长期伤残、不可逆损伤、明显疤痕或永久束缚代价后更新
        - 大脑重置达到严重惩罚并造成不可逆后果时，应同步记录
        - 没有永久损伤时保持 无

    持有金钱:
      type: number
      check:
        - 完成任务报酬、消费、交易、抢夺、赔偿后更新
        - 支出时减少，收入时增加
        - 金钱可为0，不得无理由为负

    居住地:
      type: string
      check:
        - 仅在长期住所、宿舍、据点发生改变时更新

    公开身份:
      type: string[]
      check:
        - 必须写成字符串数组，如 ["高专学生", "四级咒术师"]
        - 新身份公开获得时追加到数组
        - 身份失效、被隐藏、被剥夺时从数组中删除对应字符串
        - 禁止写成对象布尔表，如 {"高专学生": true}

    咒力:
      type: |-
        {
          总量: number,
          当前值: number,
          咒力操纵精度: number,
          有效总量: number,
          每轮回复量: number,
          消耗倍率: number
        }
      check:
        - 总量代表原始咒力上限，长期成长、升级奖励、特殊事件才改变
        - 当前值在使用术式、咒术攻击、领域展开、治疗、休息、回复后更新
        - 咒力操纵精度范围通常为 0.1 ~ 2.0；六眼持有者允许突破常规上限
        - 有效总量、每轮回复量、消耗倍率由变量结构自动计算，禁止直接修改
        - 天与咒缚（纯肉体）会让有效总量自动视为0
        - 天与咒缚（咒力异化）会让有效总量自动变为总量×3
        - 六眼会让消耗倍率自动变为0.001
        - 当前值不得超过有效总量，变量结构会自动钳制

    战斗面板:
      type: |-
        {
          最大生命值: number,
          当前血量: number,
          防御: number,
          总BP: number,
          战力等级: string,
          总肉体值_BPA: number,
          术式强度_BPB: number,
          咒术攻击: number,
          物理攻击: number,
          咒具值: number,
          伤害补正: number
        }
      check:
        - 当前血量在受伤、治疗、恢复、休息后更新
        - 咒具值在装备、失去或更换咒具时更新；四级20、三级40、二级60、一级80、特级100
        - 最大生命值、防御、总BP、战力等级、总肉体值_BPA、术式强度_BPB、咒术攻击、物理攻击、伤害补正均为派生字段，禁止直接修改
        - 当前血量不得超过 最大生命值，变量结构会自动钳制
        - 战斗公式基于新版战斗属性和输出判定体系自动结算

    基础肉体:
      type: |-
        {
          基础肉体值: number,
          武艺值: number,
          武艺阶段: string,
          武艺输出比例: number,
          武艺防御系数: number
        }
      check:
        - 基础肉体值只有长期严苛训练、体质改变、永久损伤后更新
        - 基础肉体值参考：普通人50、四级100、三级150、二级200、一级250、特级350、轮外500+
        - 武艺值范围为 0 ~ 200，战斗或专项训练后缓慢提升
        - 武艺值参考：50初学乍练、100登堂入室、150出类拔萃、180登峰造极、200出神入化
        - 武艺阶段、武艺输出比例、武艺防御系数由变量结构自动计算，禁止直接修改

    当前服装:
      type: |-
        {
          外套: string,
          内搭: string,
          下装: string,
          足具: string
        }
      check:
        - 明确描写更换衣物、战斗损毁、脱下或穿上衣物时更新对应槽位
        - 未变化的服装槽位不需要重复更新
        - 缺失或未穿戴可写 无

    名望:
      type: |-
        {
          正道: {
            数值: number,
            称号: string[]
          },
          邪道: {
            数值: number,
            称号: string[]
          }
        }
      check:
        - 正道/邪道数值范围均为 -100 ~ 100，变量结构会自动钳制
        - 完成对应阵营任务、公开行动、恶名传播后更新数值
        - 称号必须写成字符串数组，如 ["寂寂无名的路人", "新晋咒术师"]
        - 数值跨越特定阈值时，更新对应称号数组
        - 禁止将 称号 写成对象布尔表

    束缚:
      z.record: true
      key: 束缚名
      type: |-
        {
          代价: string,
          恢复条件: string
        }
      check:
        - 立下新束缚时，以束缚名作为键创建完整对象
        - 代价必须描述清楚，如 牺牲左眼视力、三小时内无法使用术式
        - 永久束缚的 恢复条件 写 无
        - 临时束缚的 恢复条件 写明解除条件
        - 达成恢复条件后删除该束缚键

    战技:
      z.record: true
      key: 战技名
      type: |-
        {
          熟练度: number,
          阶段: string,
          描述: string
        }
      check:
        - 首次学会战技时，以战技名作为键创建完整对象
        - 战斗中实际使用后熟练度通常 +1~5，上限100
        - 阶段由变量结构根据熟练度自动映射，原则上不需要手动修改
        - 描述用于说明战技效果、限制和表现
        - 战技废弃或失去时删除对应键

    生得术式:
      type: |-
        {
          名称: string,
          属性: string,
          潜力等级: string,
          潜力值: number,
          精通等级: number,
          阶段: string,
          基础强度: number,
          当前强度: number,
          描述: string
        }
      check:
        - 未觉醒时 名称 保持 待觉醒
        - 觉醒后写入名称、属性、潜力值、精通等级、描述
        - 潜力值范围 1 ~ 2000，代表成长上限和术式天赋
        - 潜力值参考：20~50砥砺型、50~100标准型、100~200精英型、200~400天才型、400~800规则干涉型、800~2000编织者型
        - 无下限咒术无六眼时潜力值锁定400，有六眼时潜力值锁定800
        - 精通等级重要节点为 1、100、200、500、800
        - 潜力等级、阶段、基础强度、当前强度由变量结构自动计算，禁止直接修改

    反转术式:
      type: |-
        {
          掌握治疗: boolean,
          熟练等级: string,
          熟练系数: number,
          可外放: boolean,
          常规回复量: number,
          常规治疗消耗: number,
          熔断修复消耗: number
        }
      check:
        - 掌握反转术式治疗时，将 掌握治疗 设为 true
        - 熟练等级允许值：未掌握、初步了解运用、能够熟练运用、能够外放
        - 六眼持有者且掌握治疗时，熟练等级会自动结算为 六眼等天赋控制
        - 熟练系数、可外放、常规回复量、常规治疗消耗、熔断修复消耗为派生字段，禁止直接修改
        - 九相图使用反转术式治疗时，消耗会自动减半
        - 熔断修复消耗不享受六眼和精度减免

    术式熔断:
      type: |-
        {
          熔断中: boolean,
          大脑重置次数: number,
          咒力回复惩罚系数: number,
          术式强度惩罚系数: number,
          熔断修复时间缩短倍率: number
        }
      check:
        - 领域展开或剧情明确导致术式熔断时，将 熔断中 设为 true
        - 熔断自然结束或通过大脑重置修复后，将 熔断中 设为 false
        - 每次主动用反转术式进行大脑重置时，大脑重置次数 +1
        - 大脑重置次数仅统计单次战斗或当前连续极限操作，战斗结束后可按剧情清算或重置
        - 熔断修复时间缩短倍率通常为2，缩短72小时及以上时为4
        - 咒力回复惩罚系数、术式强度惩罚系数由变量结构自动计算，禁止直接修改
        - 第3次大脑重置开始惩罚，第4次重罚，第5次及以上接近归零并应同步更新身体状况

    扩展术式:
      z.record: true
      key: 术式名
      type: |-
        {
          熟练度: number,
          阶段: string,
          描述: string
        }
      check:
        - 学会新通用咒术、延伸术式、反转术式等能力后添加
        - 以术式名作为键创建完整对象
        - 熟练度更新规则同 战技，范围 0 ~ 100
        - 阶段由变量结构根据熟练度自动映射，原则上不需要手动修改
        - 失去或封印该术式时删除对应键

    特殊体质:
      z.record: true
      key: 体质名
      type: string
      check:
        - 触发特殊体质时，以体质名作为键，值为体质描述
        - 固定名称优先使用：天与咒缚（纯肉体）、天与咒缚（咒力异化）、六眼、黑闪眷顾者、九相图、受肉体
        - 天与咒缚（纯肉体）：咒力有效总量为0，肉体值×2，物理补正×2
        - 天与咒缚（咒力异化）：咒力有效总量×3，生命上限×0.75，伤害×1.2，术式潜力×0.3计入肉体值
        - 六眼：咒力消耗倍率0.001，咒术输出比例100%，咒力操纵精度可突破常规上限
        - 黑闪眷顾者：黑闪概率提升200%，黑闪触发时攻击值×2.5，实际触发需剧情描述
        - 九相图：赤血操术无生命值消耗，反转术式治疗消耗减半
        - 受肉体：拥有额外独立咒力池时，应在描述中写明
        - 体质消失、被解除或不再适用时删除对应键

    咒灵操术:
      z.record: true
      key: 咒灵名
      type: string
      check:
        - 拥有对应能力并成功降伏、收服或操控咒灵后添加
        - 以咒灵名作为键，值为咒灵能力与状态描述
        - 咒灵死亡、释放、失控、被夺走时删除对应键

    行囊:
      z.record: true
      key: 物品名
      type: |-
        {
          数量: number,
          描述: string
        }
      check:
        - 获得新物品时，以物品名作为键创建完整对象
        - 消耗品使用、丢失、交易后减少数量
        - 数量为0或小于0时，变量结构会自动删除该物品键
        - 描述用于记录物品用途、品质、来源或特殊效果

  任务系统:
    z.record: true
    key: 任务名
    type: |-
      {
        任务名: string,
        任务等级: string,
        类型: string,
        委托人或势力: string,
        任务描述: string,
        完成条件: string,
        失败条件: string,
        报酬: {
          金钱: number,
          名望提升值: number,
          物品: { [key:string]: number }
        }
      }
    check:
      - 接取新任务时，必须以任务名作为键建立完整任务对象
      - 任务对象内的 任务名 应与键名保持一致或清晰对应
      - 任务等级可写 特级、一级、二级、三级、四级、特殊、未知等
      - 类型可写 祓除、护卫、调查、救援、潜入、训练、正道、邪道等
      - 报酬.物品 是物品名到数量的对象，不是数组
      - 任务完成时，将报酬发放至 user.持有金钱、user.名望、user.行囊 等对应字段，并删除该任务键
      - 任务失败、取消或过期时直接删除该任务键
      - 任务进度变化但未完成时，只更新任务描述、完成条件或失败条件等相关字段

  人际档案:
    z.record: true
    key: 角色名
    type: |-
      {
        好感数值: number,
        信任度: number,
        关系阶段: string,
        欲望值: number
      }
    check:
      - 仅在首次正式接触该角色、建立可追踪关系时创建档案
      - 以角色名作为键创建完整对象
      - 好感数值与信任度范围为 -100 ~ 100，变量结构会自动钳制
      - 欲望值范围为 0 ~ 100，变量结构会自动钳制
      - 好感数值与信任度单次变化通常在 ±1~3，重大剧情可 ±10
      - 欲望值受调情或性暗示影响 +1~5，直接性接触 +10~20
      - 欲望值达到100后的清零或高潮处理，必须有明确剧情依据
      - 关系阶段应根据好感数值、信任度与剧情实际进展同步更新为描述文字
      - 角色死亡、彻底离场或不再追踪时，可删除该角色键
