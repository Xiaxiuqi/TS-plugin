# ACU Visualizer 模块迁移测试清单

本清单用于模块迁移期间确认功能与 CSS 不被破坏。只有全部模块迁移完成且全量测试通过后，才允许进入内存优化实施阶段。

## 1. 测试原则

- 测试版迁移期间不做内存优化。
- 原插件运行文件不修改。
- 原 CSS 不修改。
- 每个模块迁移后必须完成对应单模块测试。
- 所有模块迁移完成后必须完成全量回归测试。

## 2. 单模块通用测试

每个模块迁移后至少检查：

| 测试项                  | 是否通过 | 备注 |
| ----------------------- | -------- | ---- |
| 模块可被测试版入口引用  |          |      |
| 无语法错误              |          |      |
| 输入输出与原逻辑一致    |          |      |
| 不改变 localStorage key |          |      |
| 不改变数据库 API 调用   |          |      |
| 不改变 DOM class        |          |      |
| 不改变 CSS 选择器       |          |      |
| 不夹带内存优化          |          |      |
| 不夹带渲染优化          |          |      |
| 回滚方式明确            |          |      |
