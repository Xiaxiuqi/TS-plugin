# 项目进度文档

本目录管理「回响坟冢 · 状态栏 UI」项目的阶段性进度文档。与主项目文档 `./echo-tomb-card-story-ui-plan.md` 联动，仅记录阶段变更、验证证据与下一阶段计划。

## 目录约定

```
src/card_story_ui/echo_tomb_protocol_ui/docs/
├── README.md           # 本文件，进度文档总索引
└── progress/
    ├── stage-01-bootstrap-vue-skeleton.md
    ├── stage-02-...
    └── ...
```

## 命名规则

- `stage-NN-<短标识>.md`，NN 严格递增、双位补零
- 短标识用连字符号分隔的英文小写，与阶段交付内容对齐，例 `stage-02-subject-panel`

## 每份阶段文档必填块

1. **阶段目标**：于什么阶段补齐了什么能力
2. **范围与变更清单**：新增/修改/删除的关键文件与原因
3. **关键决策**：本阶段中的选型、取舍、调整
4. **验证证据**：构建输出、产物大小、手动检查点
5. **遗留问题与风险**：应修未改、潜在隔阂、调用方注意点
6. **下一阶段入口**：付主项目文档中的 TODO 项与兴起点
7. **关联 commit**：该阶段包含的提交记录（可后补）

## 与主项目文档的联动

- 主项目文档 `./echo-tomb-card-story-ui-plan.md` 始终是唯一权威架构与 TODO 源
- 阶段文档只记录**此阶段实际发生了什么**，不重复全局架构说明
- 阶段结束后，必须同步更新主文档 t1…t12 的状态表与「修订记录」部分
