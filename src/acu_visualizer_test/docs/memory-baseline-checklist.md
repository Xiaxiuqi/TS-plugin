# ACU Visualizer 内存基线测量清单

本清单用于在实施任何内存优化前建立可对照的基线，确保后续优化不会影响原功能和 CSS。

## 1. 测量前提

- 原插件文件不改动。
- 测量对象优先为原插件当前行为。
- 后续测试版 bundle 需要与同一数据集对照。
- 每次测量记录浏览器版本、页面环境、数据规模。

## 4. DevTools Memory 测试场景

### 场景 A：首次加载

1. 打开页面。
2. 等待插件表格加载完成。
3. 记录 Heap Snapshot。
4. 记录 DOM 节点数。
5. 记录 JS heap used size。

### 场景 B：连续刷新 10 次

1. 点击刷新表格 10 次。
2. 每次等待刷新完成。
3. 记录最终 Heap Snapshot。
4. 检查 DOM 节点是否持续增加。
5. 检查 detached nodes。

### 场景 C：弹窗打开关闭 10 次

分别测试：

- 设置弹窗。
- 历史弹窗。
- 单元格编辑弹窗。
- 快捷选项弹窗。
- 主题菜单。
- 刷新菜单。

记录：

- 弹窗关闭后 DOM 是否移除。
- 是否存在 detached dialog nodes。
- 事件监听数量是否增加。

### 场景 D：搜索与清除 10 次

1. 打开搜索。
2. 输入关键词。
3. 清除搜索。
4. 重复 10 次。
5. 记录 mark 节点是否残留。
6. 记录 DOM 节点是否持续增加。

### 场景 E：编辑与保存

1. 编辑单元格。
2. 打开历史。
3. 恢复历史值。
4. 保存数据库。
5. 记录 userEditMap、diffMap、pendingDeletes 是否在保存后合理清理。

### 场景 F：拖拽排序

1. 启用表格排序。
2. 拖拽标签。
3. 保存。
4. 启用行排序。
5. 拖拽行。
6. 保存。
7. 记录拖拽状态是否释放。

## 6. CSS 对照检查

每次优化后检查：

- 主题颜色是否一致。
- 夜间模式是否一致。
- 表格边框、背景、字体是否一致。
- 弹窗样式是否一致。
- 搜索高亮是否一致。
- 拖拽状态样式是否一致。
- 通知样式是否一致。

## 7. 通过标准

优化后必须满足：

- 原功能路径无回归。
- CSS 视觉无异常。
- 重复刷新后 DOM 节点不持续增长。
- 弹窗关闭后无明显 detached nodes。
- localStorage 总量下降或不增长。
- 大表格刷新峰值内存下降。
