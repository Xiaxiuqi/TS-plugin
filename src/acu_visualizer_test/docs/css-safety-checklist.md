# ACU Visualizer CSS 零破坏检查清单

本清单用于确保任何内存优化、模块拆分或样式拆分都不会破坏原插件 CSS 表现。

## 1. 禁止事项

在没有单独确认前，禁止：

- 修改 `public/acu_visualizer/acu_visualizer.js` 中的 CSS 注入内容。
- 修改 `public/acu_visualizer/acu_visualizer-test.js` 中的 CSS 注入内容。
- 删除任何原选择器。
- 删除任何 `!important`。
- 改名任何 class。
- 改名任何 CSS 变量。
- 改变样式注入顺序。
- 让测试版 CSS 覆盖正式插件 CSS。

## 2. 样式拆分前检查

拆分 CSS 前必须记录：

| 项目                                                            | 是否完成 |
| --------------------------------------------------------------- | -------- |
| 已确认 CSS 来源函数，例如 `addStyles()`                         |          |
| 已确认搜索样式来源，例如 `injectSearchStyles()`                 |          |
| 已确认快捷弹窗样式来源，例如 `injectShortcutDialogStylesOnce()` |          |
| 已记录原注入顺序                                                |          |
| 已记录 style id                                                 |          |
| 已生成原 CSS 文本 hash                                          |          |
| 已确认测试版不影响正式插件                                      |          |

## 3. 样式拆分后检查

| 项目                              | 是否通过 |
| --------------------------------- | -------- |
| 选择器数量一致或差异已记录        |          |
| `!important` 数量一致或差异已记录 |          |
| CSS 变量名一致                    |          |
| 夜间模式 class 一致               |          |
| 主题 class 一致                   |          |
| 弹窗 class 一致                   |          |
| 表格 class 一致                   |          |
| 菜单 class 一致                   |          |
| 搜索 class 一致                   |          |
| 动画 keyframes 名称一致           |          |

## 4. 视觉检查清单

需要逐项人工或截图对照：

### 4.1 表格主体

- 容器边框。
- summary 标题栏。
- 内容区域背景。
- 表头颜色。
- 单元格边框。
- 奇偶行背景。
- hover 背景。
- 高亮变更单元格。
- 用户编辑高亮。
- 待删除行样式。

### 4.2 主题与夜间模式

- 复古羊皮。
- 极夜深空。
- 现代清爽。
- 森之物语。
- 深海幽蓝。

### 4.3 弹窗

- 设置弹窗。

## 5. CSS 回滚策略

如果出现任何视觉异常：

1. 立即禁用测试版 CSS 拆分路径。
2. 回退到原 `addStyles()`、`injectSearchStyles()`、`injectShortcutDialogStylesOnce()` 输出。
3. 保留文档中的异常记录。
4. 在修复前不得同步到正式插件。

## 6. 通过标准

只有满足以下条件，CSS 相关优化才可进入下一阶段：

- 原插件运行文件未被修改，或修改经过单独确认。
- 测试版视觉与原版一致。
- 注入顺序一致。
- 样式 id 不冲突。
- 所有差异已记录并确认无影响。
